# Domain Language

หน้านี้เป็นพจนานุกรมระบบแบบอ่านง่ายของ Legal ERP Platform ใช้สำหรับแปลงคำใน
requirement ให้เป็นภาษาที่ทีมธุรกิจ ทีมเอกสาร ทีมออกแบบ และทีมพัฒนาเข้าใจตรงกัน

รูปแบบการเขียนคือ **System Term (คำเรียกง่าย)** เมื่อเจอคำครั้งแรก หลังจากนั้นใน
เนื้อหาทั่วไปให้ใช้คำไทยได้ เพื่อให้ประโยคอ่านลื่นและไม่ดูเป็นศัพท์เทคนิคเกินไป

## How To Use This Page

- ใช้ **System Term** เมื่อต้องอ้างอิง requirement, database (ฐานข้อมูล), API
  (ช่องทางที่หน้าจอใช้คุยกับระบบหลังบ้าน) หรือคุยกับทีม development
- ใช้ **คำเรียกง่าย** เมื่อต้องเขียนคำอธิบายให้ user, admin, business team หรือ
  stakeholder อ่าน
- ถ้า requirement ใช้คำว่า **Case** แต่หมายถึงแฟ้มงานหลักของระบบ ให้ใช้ **Matter
  (แฟ้มงานกฎหมาย)** แทน
- ถ้าหมายถึงคดีในศาลจริง ๆ ให้เขียนว่า **Court Case (คดีศาล)** เพื่อไม่ให้สับสน
  กับแฟ้มงานกฎหมายของระบบ

## User-Facing Terms

| System Term         | Simple Thai            | Meaning                                    | Example                                                          |
| ------------------- | ---------------------- | ------------------------------------------ | ---------------------------------------------------------------- |
| `Matter`            | แฟ้มงานกฎหมาย          | งานกฎหมายหนึ่งเรื่องที่ระบบใช้เป็นแฟ้มหลัก | แฟ้มงานจดทะเบียนบริษัทให้ลูกความ A หรือแฟ้มคดีแรงงานของลูกความ B |
| `Client`            | ลูกความ / ผู้รับบริการ | คนหรือองค์กรที่รับบริการทางกฎหมาย          | บริษัท A เป็นลูกความ และมีหลายแฟ้มงานกฎหมายในระบบ                |
| `Document`          | เอกสาร                 | เอกสารที่สร้าง อัปโหลด หรือเก็บในระบบ      | สัญญา หนังสือมอบอำนาจ หนังสือทวงถาม หรือเอกสารประกอบคดี          |
| `Document Template` | แม่แบบเอกสาร           | แบบฟอร์มที่ระบบใช้ช่วยสร้างเอกสารใหม่      | แม่แบบหนังสือมอบอำนาจที่ดึงชื่อลูกความและเลขแฟ้มงานมาเติม        |
| `Task`              | งานย่อย                | งานที่มอบหมายให้คนหรือทีมรับผิดชอบ         | ตรวจร่างสัญญา เตรียมเอกสาร หรือโทรติดตามลูกความ                  |
| `Calendar Activity` | นัดหมาย / กำหนดเวลา    | วันนัด กิจกรรม หรือกำหนดส่งงาน             | นัดพบลูกความ กำหนดส่งเอกสาร หรือวันครบกำหนดชำระ                  |

## Commercial Terms

| System Term          | Simple Thai              | Meaning                                                                     | Example                                         |
| -------------------- | ------------------------ | --------------------------------------------------------------------------- | ----------------------------------------------- |
| `Quotation`          | ใบเสนอราคา               | ข้อเสนอค่าบริการสำหรับงานกฎหมายหรือชุดบริการ                                | ใบเสนอราคางานจดทะเบียนบริษัท 30,000 บาท         |
| `Invoice`            | ใบแจ้งหนี้               | เอกสารเรียกเก็บเงินที่พร้อมให้ลูกความชำระ                                   | ใบแจ้งหนี้งวดแรกหลังใบเสนอราคาถูกอนุมัติ        |

| `Billing`            | การวางบิลและติดตามชำระ   | การติดตามว่าออกใบแจ้งหนี้แล้วหรือยัง ครบกำหนดเมื่อไร และรับเงินแล้วหรือยัง  | ทีมการเงินดูว่างานใดยังรอชำระ                   |

## Writing Examples

ควรเขียน:

- `Matter` (แฟ้มงานกฎหมาย) คือพื้นที่รวมข้อมูลของงานกฎหมายหนึ่งเรื่อง
- หลังจากเปิดแฟ้มงานกฎหมาย ผู้ใช้งานควรเห็นลูกความ เอกสาร งานย่อย นัดหมาย
  ใบเสนอราคา และสถานะการวางบิลได้ในที่เดียว
- `Permission` (สิทธิ์การใช้งาน) ใช้กำหนดว่าผู้ใช้งานแต่ละบทบาทดู แก้ไข อนุมัติ
  หรือส่งออกข้อมูลใดได้บ้าง

ควรหลีกเลี่ยง:

- เขียน `Matter` ซ้ำทุกประโยคโดยไม่จำเป็น
- ใช้คำว่า **Case** แทนแฟ้มงานหลักของระบบ
- ใช้คำว่า **Customer** แทนลูกความหรือผู้รับบริการทางกฎหมาย

## Avoided Terms

- ใช้ **Matter** แทน **Case** เมื่อกล่าวถึงแฟ้มงานหลักของระบบ
- ใช้ **Court Case (คดีศาล)** เมื่อหมายถึงคดีในศาลจริง ๆ
- ใช้ **Client** แทน **Customer** สำหรับผู้รับบริการทางกฎหมาย
- ใช้ **Tenant (องค์กรผู้ใช้งาน)** แทน **Office Account** หรือ **BU**
- ใช้ **Finance Foundation (ข้อมูลตั้งต้นด้านการเงิน)**
  เมื่อหมายถึงข้อมูลการเงินตั้งต้น ไม่ใช่ระบบบัญชีเต็มรูปแบบ
