# Domain Language

หน้านี้กำหนดคำศัพท์กฎหมายที่ต้องมีคำไทยกำกับในเอกสารของ Legal ERP Platform
เพื่อให้ทีมธุรกิจ ทีมเอกสาร ทีมออกแบบ และทีมพัฒนาเข้าใจตรงกันเมื่ออ่าน
เอกสารระบบ

กติกาหลักคือใส่วงเล็บคำไทยเฉพาะศัพท์กฎหมาย หรือคำที่ต้องเข้าใจในบริบทงานกฎหมาย
โดยตรง ส่วนคำระบบ คำธุรกิจ และคำเทคนิคทั่วไปให้คงคำอังกฤษไว้ แล้วอธิบายเป็น
ภาษาไทยเฉพาะเมื่อจำเป็น

## How To Use This Page

- ใช้ **Legal Term (คำไทย)** เมื่อกล่าวถึงศัพท์กฎหมายครั้งแรกในแต่ละหน้า
- หลังจากแนะนำศัพท์กฎหมายแล้ว สามารถใช้คำไทยต่อได้ เช่น “แฟ้มงานกฎหมาย”
  แทนการเขียน `Matter` ซ้ำทุกประโยค
- คำที่ไม่ใช่ศัพท์กฎหมาย เช่น `Tenant`, `Role`, `Permission`, `Quotation`,
  `Invoice`, `API` หรือ `Database` ไม่ต้องใส่วงเล็บคำแปลไทย
- ถ้าพบคำว่า **Case** แต่หมายถึงแฟ้มงานหลักของระบบ ให้ใช้ **Matter
  (แฟ้มงานกฎหมาย)** แทน
- ถ้าหมายถึงคดีในศาลจริง ๆ ให้เขียนว่า **Court Case (คดีศาล)** เพื่อไม่ให้สับสน
  กับแฟ้มงานกฎหมายของระบบ

## Legal Terms

| Legal Term          | Thai Term              | Meaning                                    | Example                                                          |
| ------------------- | ---------------------- | ------------------------------------------ | ---------------------------------------------------------------- |
| `Matter`            | แฟ้มงานกฎหมาย          | งานกฎหมายหนึ่งเรื่องที่ระบบใช้เป็นแฟ้มหลัก | แฟ้มงานจดทะเบียนบริษัทให้ลูกความ A หรือแฟ้มคดีแรงงานของลูกความ B |
| `Client`            | ลูกความ / ผู้รับบริการ | คนหรือองค์กรที่รับบริการทางกฎหมาย          | บริษัท A เป็นลูกความ และมีหลายแฟ้มงานกฎหมายในระบบ                |
| `Court Case`        | คดีศาล                 | คดีที่อยู่ในกระบวนการศาลจริง ๆ             | คดีแรงงานที่มีนัดพิจารณาในศาล                                    |
| `Document`          | เอกสารทางกฎหมาย        | เอกสารที่เกี่ยวข้องกับงานกฎหมาย            | สัญญา หนังสือมอบอำนาจ หนังสือทวงถาม หรือเอกสารประกอบคดี          |
| `Document Template` | แม่แบบเอกสารทางกฎหมาย  | แบบฟอร์มที่ระบบใช้ช่วยสร้างเอกสารกฎหมาย    | แม่แบบหนังสือมอบอำนาจที่ดึงชื่อลูกความและเลขแฟ้มงานมาเติม        |
| `Legal Service`     | บริการทางกฎหมาย        | ประเภทบริการที่สำนักงานกฎหมายให้ลูกความ    | งานจดทะเบียนบริษัท งานร่างสัญญา หรืองานคดีแรงงาน                 |

## Writing Examples

ควรเขียน:

- `Matter` (แฟ้มงานกฎหมาย) คือพื้นที่รวมข้อมูลของงานกฎหมายหนึ่งเรื่อง
- หลังจากเปิดแฟ้มงานกฎหมาย ผู้ใช้งานควรเห็น `Client` (ลูกความ/ผู้รับบริการ),
  `Document` (เอกสารทางกฎหมาย), task, quotation และสถานะ billing ได้ในที่เดียว
- `Court Case` (คดีศาล) ใช้เฉพาะเมื่อหมายถึงคดีที่อยู่ในกระบวนการศาลจริง ๆ
- `Permission` ใช้กำหนดว่าผู้ใช้งานแต่ละ role ดู แก้ไข อนุมัติ หรือส่งออกข้อมูล
  ใดได้บ้าง

ควรหลีกเลี่ยง:

- เขียน `Matter` ซ้ำทุกประโยคโดยไม่จำเป็น
- ใช้คำว่า **Case** แทนแฟ้มงานหลักของระบบ
- ใช้คำว่า **Customer** แทนลูกความหรือผู้รับบริการทางกฎหมาย
- ใส่วงเล็บคำแปลไทยให้คำระบบทุกคำ เช่น `Tenant` หรือ `Permission` ในทุกหน้า

## Avoided Terms

- ใช้ **Matter** แทน **Case** เมื่อกล่าวถึงแฟ้มงานหลักของระบบ
- ใช้ **Court Case (คดีศาล)** เมื่อหมายถึงคดีในศาลจริง ๆ
- ใช้ **Client** แทน **Customer** สำหรับผู้รับบริการทางกฎหมาย
- ใช้ **Tenant** แทน **Office Account** หรือ **BU**
- ใช้ **Finance Foundation** เมื่อหมายถึงข้อมูลการเงินตั้งต้น
  ไม่ใช่ระบบบัญชีเต็ม รูปแบบ
