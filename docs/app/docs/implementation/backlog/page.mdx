## Purpose

แปลง requirement และ SOP เป็น implementation items ที่มีลำดับ dependency,
acceptance boundary และ UAT coverage ชัดเจน โดยไม่รวมรายละเอียด task ระดับโค้ด
ที่ทีมจะประเมินหลังเลือก sprint/release

## Priority and Readiness

| Priority  | Meaning                                                    |
| --------- | ---------------------------------------------------------- |
| P0        | Platform/security/control ที่ domain อื่นต้องพึ่งพา        |
| P1        | Legal Core และ workflow ที่จำเป็นต่อ Matter operation      |
| P2        | Commercial, billing และ assurance ที่ต่อยอด Legal Core     |
| P3        | Optimization/integration หลัง control หลักผ่าน UAT         |
| Discovery | Requirement อยู่ใน scope แต่ Decision/SOP ยังไม่พอให้พัฒนา |

Definition of Ready: Requirement ID, SOP/Decision, acceptance criteria, data
owner, permission, audit event, dependency และ UAT case ครบ

Definition of Done: implementation/review/test ผ่าน, migration/rollback พร้อม,
UAT mapped cases Passed และ release evidence ถูกเก็บ

### P0 Acceptance Boundary

- Tenant ID มาจาก authenticated context ห้ามเชื่อ client-supplied scope
  เพียงอย่างเดียว
- Permission decision ตรวจ action + Tenant + team/Matter/record scope ทุกครั้ง
- Approval และ event write อยู่ transaction เดียวกับ business change หรือใช้
  outbox ที่ตรวจ reconciliation ได้
- Event ต้นฉบับแก้ไม่ได้ Correction ต้องเชื่อม original และ current
  interpretation
- Scheduler retry แบบ idempotent และไม่ auto-approve เมื่อ SLA เกิน

## Delivery Sequence

1. P0 Tenant/RBAC/Audit/Workflow foundation
2. Client + Matter intake/lifecycle
3. Document + Task/Calendar
4. Quotation + Invoice/Payment
5. Reports + Retention + Governance automation
6. Discovery domains หลัง SOP/Decision/UAT พร้อม

## Backlog Controls

- Backlog ID ไม่ reuse และทุก status change มี actor/time/reason
- Dependency ต้องเสร็จหรือมี accepted technical stub ก่อนเริ่ม downstream item
- UAT case ต้อง map ได้เพียงพอทั้ง positive, negative, authority และ exception
- Scope/authority change ต้องอัปเดต Traceability, Mapping, RBAC, Workflow และ
  UAT
- Item ที่มี cross-Tenant, financial posting, Legal Hold หรือ audit integrity
  risk ห้าม release ด้วย known P1/P2 defect
