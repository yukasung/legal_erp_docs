## Purpose

กำหนด permission code ระดับ action และ baseline grant ของ Admin, Lawyer,
Assistant, Finance, HR และ Manager โดย approval authority ยังคงผ่าน workflow
ไม่ใช่เพียงมี permission แล้วอนุมัติได้ทุก record

## Evaluation Model

คำขอผ่านเมื่อเงื่อนไขทั้งหมดเป็นจริง:

1. Membership Active และ Tenant ตรง authenticated context
2. Package/module entitlement เปิดใช้งาน
3. Role/individual grant มี action permission และยังไม่หมดอายุ
4. Resource scope ตรง Tenant, team, Matter, owner หรือ record restriction
5. Workflow authority/SoD อนุญาต actor สำหรับ request ปัจจุบัน
6. Record status อนุญาต action และไม่มี Legal Hold/security block ที่เกี่ยวข้อง

UI visibility ไม่ใช่ security control API ต้องประเมินครบทุกข้อทุกครั้ง

| Value   | Meaning                                                       |
| ------- | ------------------------------------------------------------- |
| Full    | ทำ action ได้ภายใน Tenant และ functional scope ที่มอบหมาย     |

| Execute | ดำเนินการหลัง approval ไม่มี business decision authority      |
| Read    | อ่านตาม data scope                                            |
| No      | ไม่มี baseline grant                                          |

## Permission Object

Grant ต้องเก็บ seed_id, permission_code, subject membership/bundle, effect,
scope_code + scope_reference, condition_code, effective_at, expires_at, reason,
request/approval references, granted_by และ revoked_at/by ห้ามใช้ free-text role
name หรือข้อความใน human-readable matrix ตัดสินสิทธิ์ใน business code

## Scope Expression Vocabulary

| Scope Code        | Enforced Predicate                                                        |
| ----------------- | ------------------------------------------------------------------------- |
| TENANT            | resource.tenant_id = context.tenant_id                                    |
| TEAM              | Tenant + resource.team_id in active user teams                            |
| ASSIGNED_MATTER   | Tenant + active matter membership/assignment                              |
| OWN_RECORD        | Tenant + resource.owner_id = current membership                           |
| FINANCE_DOMAIN    | Tenant + finance/billing resource; excludes legal content fields          |
| HR_DOMAIN         | Tenant + personnel/payroll/leave resource; excludes Client/Matter/Billing |
| EXPLICIT_RECORD   | Tenant + active individual grant for resource type/id                     |
| CONFIG_ONLY       | Tenant configuration metadata; excludes business record content           |
| EMERGENCY_SESSION | exact approved resource/actions until expires_at; all access logged       |

Scope compiler รับเฉพาะ code/typed reference จาก catalog นี้ ห้ามรับ SQL/free
text

## Toxic Combinations

| Combination                                  | Control                                                    |
| -------------------------------------------- | ---------------------------------------------------------- |
| prepare + approve quotation เดียวกัน         | runtime self-approval block                                |
| assess High risk + A1/A2 decision            | assessor cannot approve same decision                      |
| request privileged access + A1/A2 + execute  | three-party separation where available; never self-approve |
| create financial adjustment + approve + post | maker-checker and executor event                           |
| prepare disposal + A1/A2 + execute           | dual approval and separate executor                        |
| Admin support + business approve             | Acting Role cannot add business authority                  |

## Denial and Audit Rules

- API ตอบผลทั่วไปที่ไม่เปิดเผยว่ามี cross-Tenant record อยู่หรือไม่
- บันทึก denied event สำหรับ sensitive view/export, approval, privilege และ
  cross-scope action
- Event เก็บ policy/rule version และ matched/missing permission โดยไม่เก็บ
  secret
- Permission cache invalidate เมื่อ grant/revoke/role/scope/status เปลี่ยน
- Break-glass session แสดง banner, จำกัดเวลา, log ทุก view/change/export และ
  review หลังใช้
