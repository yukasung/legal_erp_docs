## Purpose

technical mapping และ UAT เพื่อแยก capability ที่พร้อม development ออกจาก
capability ที่ต้องปิด policy/control gap ก่อน

## Gap 1: Time Tracking & Timesheets

Required decisions before development:

- timer vs manual entry lifecycle, rounding/minimum unit and timezone/date rules
- billable/non-billable classification, activity/rate source and Matter status
  gate
- submit/approve/reject/reopen/correct authority and self-approval
- locked billing period, invoice linkage and correction after invoicing
- utilization/report definitions, retention and audit evidence

Required artifacts: DEC-TIM series, SOP-TIM-001, database/API/UI mapping
extension, RBAC actions, workflow config, fixtures and UAT cases

## Gap 2: Finance Foundation

Invoice/Payment SOP ครอบคลุมลูกหนี้และการรับชำระ แต่ยังไม่พอสำหรับ Finance
Foundation ทั้งระบบ ต้องกำหนด:

- chart of accounts, fiscal period, tax master and posting boundary
- receipt/payment/expense/journal source, approval and immutable posting
- bank/cash reconciliation, close/reopen period and correction
- multi-currency/exchange rate scope, withholding/VAT evidence and reports
- segregation between Finance maker, Manager approver and Admin executor

ห้ามใช้คำว่า general ledger หรือ financial statements พร้อมใช้งานจน DSC-FIN-001
ผ่าน Definition of Ready

## Gap 3: HR, Payroll and Leave

ต้องแยกสาม control domains:

| Domain    | Minimum Decisions / Controls                                                                          |
| --------- | ----------------------------------------------------------------------------------------------------- |
| Personnel | employee identity/status, organization assignment, sensitive fields, joiner/mover/leaver, retention   |
| Payroll   | earning/deduction/tax/social security, payroll period/lock, retro/correction, maker-checker, payslip  |
| Leave     | leave type/accrual/balance, request/approval/cancel, overlap/calendar, payroll effect, year-end carry |

Data boundary ต้องยืนยันว่า Finance ไม่เห็น payroll รายบุคคล และ Manager เห็นตาม
responsibility scope HR export ต้องมี purpose/recipient/approval/audit/expiry

## Gap 4: Tenant Onboarding and Package

ต้องกำหนด signup/verification, domain ownership, initial Tenant Admin,
package/module entitlement, user limit, upgrade/downgrade effective time,
over-limit remediation, trial/suspension/termination, data export/retention และ
platform support boundary

Platform subscription billing ยังอยู่นอก scope เริ่มต้นและต้องไม่ปนกับ billing
ที่ สำนักงานออกให้ Client

## Gap 5: Production Integrations

Core implementation ใช้ adapter/stub ได้ แต่ก่อนเปิด production ต้องมี provider
contract สำหรับ:

- e-signature identity, signer consent, evidence bundle, webhook/replay and
  outage
- calendar/email sender identity, consent/token storage, rate limit and retry
- object storage encryption, malware scan, regional/retention and signed URL
- export delivery recipient authentication, expiry and revocation

Provider selection ไม่เปลี่ยน business status/authority โดยตรง หาก provider
capability ต่ำกว่า SOP control ให้เปลี่ยน provider หรือเพิ่ม compensating
control

## Gap Control Rules

- Gap owner เปิด Decision/SOP work item ก่อน feature implementation
- Prototype ใช้ synthetic data และห้ามสร้าง production authority โดยไม่มี
  control
- Gap closure ต้องอัปเดต Scope, Traceability, Master Register, Backlog, Mapping,
  RBAC, Workflow, Fixtures และ Release Checklist
- Rating เปลี่ยนเป็น Ready เมื่อ review evidence, authored UAT และ mapping
  ครบตาม Definition of Ready; การเริ่ม development ยังต้องผ่าน dependency gate
  และการ release ต้องให้ mapped UAT execution ผ่าน ไม่ใช่เพียงมี UI
