## Purpose

กำหนด deterministic synthetic data, user/role setup, seed order, fixture deltas,
expected controls และ cleanup สำหรับ UAT 69 cases โดยไม่ใช้ข้อมูลบุคคลจริง

## Environment Rules

- Environment code UAT และ build/release ID ต้องปรากฏใน Test Run
- Tenant UAT-A และ UAT-B ใช้ encryption/storage namespace แยกเหมือน production
- Email ใช้ example.test, identifier ใช้ prefix UAT และห้ามส่ง notification
  ออกภายนอก
- Time ถูกควบคุมด้วย injectable clock สำหรับ SLA/expiry/overdue cases
- Integration ใช้ sandbox/stub ที่บันทึก request, signature, response, retry และ
  webhook
- Seed manifest มี version/checksum และ rerun แบบ idempotent

## Seed Order

กรณีที่ระบุ sub-fixture A/B ต้อง materialize เป็น record แยกพร้อม resolved
payload/checksum ของตนเอง และ test run ต้อง execute/assert ทั้งสอง sub-fixture
ห้ามสุ่มเลือก branch ระหว่าง run

1. Clock/calendar, currencies, countries, tax and canonical status master
2. Tenant, domain, package, entitlement and user limit
3. Users, memberships, MFA test methods, roles and permission grants
4. Clients, identifiers, interactions and Retainer fixtures
5. Matters, parties, members, assessments, decisions and transitions
6. Documents/files/checksums, tasks, appointments and integration connections
7. Quotations, invoices, payments, allocations and expected report source data
8. Audit chain, retention rules, Legal Holds, exceptions and controlled
   artifacts
9. Generate manifest with record IDs, versions, checksums and expected totals

Seed ล้มเหลวต้อง rollback ทั้ง run หรือระบุ partial run ID ที่ cleanup ได้
ห้ามใช้ production migration สร้าง UAT-only users/records

## Reset and Evidence

- Test Run สร้าง namespace/run_id และ clone fixture; teardown ลบเฉพาะ namespace
  นั้น
- Immutable evidence ที่ต้องตรวจหลัง run เก็บจน QA sign-off แล้วจึงใช้ UAT
  retention rule
- Reset ยืนยันว่า scheduled jobs, tokens, share links และ provider callbacks ถูก
  revoke
- Evidence package เก็บ build, manifest checksum, Step Set, FX ID, actor,
  actual/expected, database/event references และ defect/exception ID
- Cross-Tenant cases ตรวจทั้ง content, count, error message, export, object key
  และ audit metadata
