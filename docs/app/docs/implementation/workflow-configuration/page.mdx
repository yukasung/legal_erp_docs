## Purpose

กำหนด workflow configuration contract สำหรับ lifecycle, approval, SLA,
escalation และ notification เพื่อให้ business rules มี version/effective date
และไม่ถูก hard-code กระจายหลาย service

## Configuration Contract

| Field             | Requirement                                                         |
| ----------------- | ------------------------------------------------------------------- |
| workflow_code     | ID ไม่ซ้ำ เช่น MATTER_LIFECYCLE                                     |
| version           | เพิ่มเมื่อ state, guard, authority, SLA หรือ notification เปลี่ยน   |
| tenant_scope      | global baseline หรือ Tenant override ที่อนุมัติ                     |
| effective_from/to | ช่วงเวลาที่ rule ใช้; ห้าม overlap version ใน scope เดียวกัน        |
| states            | canonical value, terminal flag, entry/exit conditions               |
| transitions       | from, to, command, guard set, authority rule, side effects          |
| approval_rule     | requester, A1/A2, executor, self-approval/SoD rule                  |
| sla_rule          | start/pause/resume/stop, duration, calendar, warning, breach action |
| notification_rule | trigger, audience, channel, template, retry, deduplication          |
| evidence_rule     | required fields/attachments/events and retention class              |
| status            | Draft, In Review, Approved, Superseded, Retired                     |

Approved instance ต้อง snapshot workflow/version/authority ที่ใช้ไว้กับ request
เพื่อไม่ให้ rule change ย้อนหลังเปลี่ยนความหมายของ decision เดิม

## Matter Lifecycle

| From             | Command / To                    | Required Guard                                                  | Authority                                                                                      | Side Effects                                      |
| ---------------- | ------------------------------- | --------------------------------------------------------------- | ---------------------------------------------------------------------------------------------- | ------------------------------------------------- |
| Intake Draft     | submit-intake → Conflict Review | Client exists, intake required fields, duplicate disposition    | Lawyer/Assistant request; authorized user/system execute                                       | start conflict SLA, notify assessor               |
| Intake Draft     | cancel → Cancelled              | cancellation reason                                             | Lawyer/Assistant request; Manager approve; authorized user/system execute                      | terminal notice, preserve prior reference         |
| Conflict Review  | clear → Risk Review             | latest Conflict Cleared                                         | Responsible Lawyer; no approval when Cleared, Manager if previously Escalated                  | stop conflict SLA, start risk SLA                 |
| Conflict Review  | prohibit → Rejected             | Conflict Prohibited                                             | Responsible Lawyer actor; no additional approval/no override; System execute                   | terminal notice, preserve evidence                |
| Risk Review      | submit → Pending Approval       | Risk rating/evidence complete                                   | assessor request                                                                               | start acceptance SLA, assign A1/A2                |
| Pending Approval | approve → Active                | Acceptance Approved, Commercial Readiness, Client/Lawyer Active | Low/Medium Manager A1; High Manager A1 + Manager A2                                            | activate access/work, notify team                 |
| Pending Approval | reject → Rejected               | existing Acceptance Rejected decision/reason                    | Acceptance Approver actor; no duplicate approval; System execute                               | terminal notice                                   |
| Active           | hold → On Hold                  | reason and review date                                          | Responsible Lawyer request; Manager approve; authorized user/system execute                    | pause allowed SLA, notify team/client as policy   |
| Active           | immediate-hold → On Hold        | legal/sanctions/ethical/data security/preservation reason       | Responsible Lawyer or Admin actor; immediate execute; Manager review due in 1 business day     | stop affected work immediately, open review clock |
| On Hold          | resume → Active                 | hold reason resolved, reassessment if trigger                   | Responsible Lawyer request; Manager approve; authorized user/system execute                    | resume clocks, notify owners                      |
| Active           | complete → Completed            | substantive work complete                                       | Responsible Lawyer request; Manager approve; authorized user/system execute                    | run closure dependencies                          |
| Completed        | close → Closed                  | task/document/time/billing checklist clear                      | Lawyer/Assistant request; Manager approve; authorized user/system execute                      | lock normal commands, retention trigger           |
| Completed        | reactivate → Active             | reason to continue work                                         | Responsible Lawyer request; Manager approve; authorized user/system execute                    | resume work, create reopening timeline event      |
| Closed           | reopen → Conflict Review        | reopen reason and Manager approval                              | Responsible Lawyer request; Manager approve; alternate Manager if requester holds Manager role | recheck conflict/risk, new timeline event         |

Rejected/Cancelled terminal states ห้าม reopen; ต้องสร้าง Matter ใหม่ที่เชื่อม
original reference เมื่อ policy อนุญาต

Immediate Hold คงสถานะ On Hold จน Manager บันทึกผล review; การไม่ยืนยันภายใน 1
business day ต้อง breach/escalate แต่ห้าม resume อัตโนมัติ

## Document Workflow

| State             | Allowed Command   | Next State        | Guard / Authority                                     |
| ----------------- | ----------------- | ----------------- | ----------------------------------------------------- |
| Draft             | submit-review     | In Review         | required metadata/version/checksum; reviewer assigned |
| In Review         | request-changes   | Changes Requested | reviewer reason required                              |
| Changes Requested | create-version    | Draft             | new version; reviewed version unchanged               |
| In Review         | approve           | Approved          | Senior Lawyer/Manager; no self-approval               |
| Approved          | request-signature | Pending Signature | signer/order/expiry valid                             |
| Pending Signature | complete-signing  | Signed            | provider/identity evidence and final checksum         |
| Approved/Signed   | supersede         | Superseded        | replacement approved/signed version                   |
| Any non-terminal  | expire/cancel     | Expired/Cancelled | authority/reason; revoke active shares                |

External share เป็น workflow ย่อย: request →
classification/recipient/purpose/expiry validation → Manager A1 และ
Restricted/Privileged Admin A2 → issue link → expire/revoke → retain access log

Admin A2 ต้องเป็นคนละบัญชีกับ Admin ผู้ execute workflow เดียวกัน

## Access Workflow

| Request                        | Approval                           | Execution / Expiry                                                  |
| ------------------------------ | ---------------------------------- | ------------------------------------------------------------------- |
| Standard user/role             | Manager/Tenant Authority           | Admin invite/activate after MFA; no self-approval                   |
| Standard individual permission | Manager                            | Admin grant with reason/scope/expiry                                |
| Privileged permission          | Manager A1 + Admin A2              | Admin grant; mandatory expiry and recertification                   |
| Emergency Access               | Authorized Manager                 | Admin grant time-bound Acting Role; auto-revoke and post-use review |
| Suspend/deactivate             | Manager or Admin per reason        | revoke sessions/tokens then ownership reconciliation                |
| Recertification                | Manager; Admin confirms privileged | Admin Keep/Modify/Revoke execution and snapshot                     |

Admin A2 ต้องเป็นคนละบัญชีกับ Admin ผู้ grant/revoke/execute คำขอเดียวกัน

## Report, Retention and Exception Workflow

- Report run: Requested → Generated → Validated หรือ Variance Open → Published
- Sensitive export: Requested → A1/A2 when required → Generated → Delivered →
  Expired/Revoked
- Audit finding: Open → Assigned → Corrected/Accepted → Verified → Closed
- Legal Hold: Draft → Approved → Active → Released; Active blocks disposal
  instantly
- Disposal batch: Candidate → Hold/Dependency Checked → A1/A2 Approved →
  Executed → Certified
- Exception: Open → Contained → In Remediation → Resolved → Closed; Cancelled
  ใช้เมื่อ duplicate/invalid request และยังต้องเก็บ audit reason

## Job and Idempotency Controls

- Scheduler claim job ด้วย lock/lease และบันทึก attempt, next retry, final
  outcome
- Command ใช้ idempotency key ต่อ Tenant + action + business reference
- Notification/share/export retry ใช้ stable deduplication key
- Clock job ใช้ database time และตรวจ due window ซ้ำได้โดยไม่สร้าง breach event
  ซ้ำ
- Webhook out-of-order ต้องเปรียบเทียบ provider sequence/event time และ current
  state
- Manual retry แสดง original failure/attempts และต้องมี permission/audit reason
