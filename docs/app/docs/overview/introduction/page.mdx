# Introduction

Legal ERP Platform คือระบบบริหารจัดการงานกฎหมายแบบ matter-centered สำหรับ
สำนักงานกฎหมายและองค์กรที่ให้บริการด้านกฎหมาย ระบบนี้รวมข้อมูล client, matter,
task, document, calendar activity, quotation, billing และข้อมูลด้านการเงินไว้ใน
พื้นที่ทำงานเดียวกัน

เป้าหมายของ platform คือช่วยลดงานธุรการที่ต้องทำซ้ำ ลดความผิดพลาดของข้อมูล
client และ matter และทำให้ทีมเห็นสถานะงานตั้งแต่รับเรื่องจนถึงส่งมอบงานและเรียก
เก็บเงินได้ชัดเจนขึ้น โดยให้ matter เป็นแฟ้มงานหลักที่เชื่อมโยงบุคคล เอกสาร งาน
นัดหมาย ใบเสนอราคา ใบแจ้งหนี้ การรับชำระเงิน และข้อมูล profitability ไว้ใน
record เดียวกัน

## Who It Serves

- สำนักงานกฎหมายที่ต้องการพื้นที่ทำงานร่วมกันสำหรับบริหารงานกฎหมาย
- ทีมกฎหมายที่ต้องดูแล client, matter และ document จำนวนมาก
- ผู้ดูแลระบบที่ต้องจัดการ role, permission, tenant setting และรายงาน
- ทีมการเงินหรือทีมปฏิบัติการที่ต้องติดตามสถานะ quotation, billing และ
  collection

## Core Capabilities

- บริหารข้อมูล client และ matter
- สร้างและจัดเก็บ document จากศูนย์กลาง
- จัดการ task, team และ calendar
- ติดตาม quotation, billing และข้อมูลทางการเงิน
- จัดการ user, role, permission และ tenant setting
- วางรากฐานรายงานสำหรับวิเคราะห์การดำเนินงานและ profitability

## Operating Model

แต่ละ tenant หมายถึงหนึ่งสำนักงานกฎหมายหรือหนึ่งองค์กรที่ใช้งาน platform ใน
environment แบบ SaaS ที่แยกข้อมูลออกจาก tenant อื่น ผู้ใช้แต่ละคนทำงานผ่าน
permission ตามบทบาท เพื่อให้มองเห็นข้อมูลและทำรายการได้เฉพาะส่วนที่เกี่ยวข้องกับ
หน้าที่ของตน

matter เป็นศูนย์กลางของงานประจำวัน ข้อมูล client, document, task, calendar
event, quotation, invoice, payment และ performance data ควรถูกย้อนดูได้จาก
matter
เพื่อให้ทีมเข้าใจสถานะปัจจุบันของงานโดยไม่ต้องค้นหาข้อมูลจากหลายเครื่องมือ
ที่แยกจากกัน

## Technical Foundation

stack ที่ยึดเป็นทางการสำหรับ Legal ERP Platform มีดังนี้:

- **Backend**: Django
- **Frontend**: React และ Tailwind CSS
- **Database**: PostgreSQL
