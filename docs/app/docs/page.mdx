# Legal ERP Platform Docs

เอกสารชุดนี้ใช้เป็นพื้นที่กลางสำหรับอธิบาย requirement, scope, module, workflow,
database, API และ UI ของ Legal ERP Platform

Legal ERP Platform เป็นระบบบริหารงานกฎหมายแบบ matter-centered สำหรับสำนักงาน
กฎหมายและทีมกฎหมาย โดยใช้ `Matter` เป็น record หลักที่เชื่อม client, document,
task, calendar, quotation, billing และ finance foundation เข้าด้วยกัน

## Start Here

- [Overview](/docs/overview): ภาพรวมของ platform และแผนที่เอกสาร
- [Scope](/docs/overview/scope): ขอบเขต baseline สิ่งที่อยู่ใน scope
  สิ่งที่อยู่นอก scope และคำถามที่ต้องตัดสินใจต่อ
- [Architecture](/docs/overview/architecture): พื้นที่สำหรับสรุปสถาปัตยกรรมระบบ
  และ component หลัก

## Documentation Structure

- **Overview**: ภาพรวมระบบ เป้าหมาย กลุ่มผู้ใช้ ความสามารถหลัก และ technical
  foundation
- **Access**: role, permission และหลักการควบคุมสิทธิ์การใช้งาน
- **Modules**: รายละเอียด module หลัก เช่น clients, matters, tasks, documents,
  billing และ finance
- **Process**: workflow และ flow การทำงานข้าม module
- **Technical**: database, API และ UI specification
- **Changes**: changelog และบันทึกการเปลี่ยนแปลงของเอกสาร

## Current Focus

ตอนนี้เอกสารกำลังเริ่มจากภาพรวมระบบก่อน เพื่อให้คำศัพท์ ขอบเขต และโครงสร้างหลัก
ชัดเจนก่อนลงรายละเอียดราย module

ลำดับถัดไปที่เหมาะสมคือทำหน้า Architecture แล้วค่อยแตกลง Database, API และ UI
ตาม requirement ต้นทาง
