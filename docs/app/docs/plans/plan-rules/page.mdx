# Plan Rules

หน้านี้กำหนดกฎสำหรับการเปิดฟีเจอร์ การเปลี่ยนแผน และการรักษาข้อมูลขององค์กร
โดยแยกสิทธิ์ตามแผนออกจาก Role และ Permission ของผู้ใช้

## Plan Inheritance

STANDARD → ESSENTIALS → PRO → ADVANCED → ENTERPRISE

- แต่ละแผนรวมความสามารถที่เปิดในแผนก่อนหน้า
- Not Included จะเปิดได้เมื่อแผนระดับถัดไประบุเป็น Included
- View Only ไม่อนุญาตให้สร้าง แก้ไข ลบ อนุมัติ หรือส่งออก
- TBD ไม่สืบทอดเป็น Included และห้ามเปิดจนกว่าจะมีการอนุมัติ
- Baseline ต้องเปิดทุกแผน โดยเฉพาะการรักษาความปลอดภัยและ Audit

## Feature Access Rules

ระบบอนุญาตให้ใช้งานเมื่อผ่านทุกเงื่อนไขต่อไปนี้:

1. องค์กรอยู่ในสถานะใช้งานและแผนยังมีผล
2. แผนและเวอร์ชันขององค์กรเปิดโมดูลหรือฟีเจอร์
3. บัญชีผู้ใช้งานยัง Active และผ่านนโยบายการยืนยันตัวตน
4. Role หรือ Permission อนุญาตให้ดำเนินการ
5. ข้อมูลอยู่ภายในองค์กร ทีม แฟ้มงาน หรือขอบเขตที่ได้รับมอบหมาย
6. สถานะรายการและระดับอนุมัติรองรับการดำเนินการนั้น

หากเงื่อนไขใดไม่ผ่าน ระบบต้องปฏิเสธการดำเนินการโดยไม่เปิดเผยข้อมูลนอกขอบเขต
และบันทึกเหตุผลสำหรับการตรวจสอบย้อนหลัง

## Upgrade Rules

- ตรวจสอบแผนปัจจุบัน แผนใหม่ และวันที่มีผลก่อนดำเนินการ
- เปิดเฉพาะฟีเจอร์ที่แผนใหม่ระบุว่า Included, View Only หรือ Baseline
- ไม่เพิ่ม Role, Permission หรืออำนาจอนุมัติให้ผู้ใช้โดยอัตโนมัติ
- ทดสอบฟีเจอร์ที่เปิดใหม่และบันทึกแผนเดิม แผนใหม่ ผู้อนุมัติ และผู้ดำเนินการ
- แจ้งผู้ดูแลระบบขององค์กรเมื่อการเปลี่ยนแผนมีผล

## Downgrade Rules

- ห้ามลบข้อมูล เอกสาร ประวัติรายการ หรือ Audit Log โดยอัตโนมัติ
- ปิดการสร้างหรือแก้ไขข้อมูลในฟีเจอร์ที่แผนใหม่ไม่รองรับ
- รักษาความสัมพันธ์ของข้อมูลเดิมและความสามารถในการตรวจสอบย้อนหลัง
- ตรวจสอบงานค้าง รายการที่ยังไม่อนุมัติ และการเชื่อมต่อก่อนเปลี่ยนแผน
- กำหนดแนวทางดูข้อมูลเดิม การส่งออก ระยะผ่อนผัน และการกลับมาใช้แผนเดิม ก่อนนำ
  Downgrade ไปใช้จริง

## Baseline Controls

ความสามารถต่อไปนี้ห้ามลดระดับตามราคาแผน:

- การแยกข้อมูลของแต่ละองค์กร
- การเข้ารหัสและการยืนยันตัวตน
- Role-Based Access Control และการจำกัดขอบเขตข้อมูล
- Audit Trail สำหรับเหตุการณ์สำคัญ
- Retention, Legal Hold และการทำลายข้อมูลตามขั้นตอน
- Backup, Recovery และการตรวจสอบความสมบูรณ์ของข้อมูล

## Change Evidence

ทุกการเปลี่ยนแผนต้องบันทึกข้อมูลต่อไปนี้:

|     รายการ     | ข้อมูลที่ต้องเก็บ                                        |
| :------------: | -------------------------------------------------------- |
|  Organization  | องค์กรและ Tenant ที่ได้รับผลกระทบ                        |
|  Plan Change   | แผนเดิม แผนใหม่ เวอร์ชัน และวันที่มีผล                   |
|    Approval    | ผู้ขอ ผู้อนุมัติ และเหตุผล                               |
|   Execution    | ผู้ดำเนินการ เวลา และผลการดำเนินการ                      |
| Feature Result | ฟีเจอร์ที่เปิด ปิด หรือเปลี่ยนเป็น View Only             |
| Reconciliation | งานค้าง จำนวนผู้ใช้ ข้อมูล และการเชื่อมต่อที่ตรวจสอบแล้ว |
