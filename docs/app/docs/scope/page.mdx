# Scope

หน้านี้กำหนดว่า baseline ของ Legal ERP Platform รวมอะไร ไม่รวมอะไร และมีคำถาม
ใดที่ต้องตัดสินใจต่อก่อนลงรายละเอียด module, workflow, database, API และ UI

เอกสารต้นทางบางส่วนใช้คำว่า `Case Management` แต่ในเอกสารนี้จะใช้คำว่า `Matter`
เป็นหลัก เพื่อให้ตรงกับ [Domain Language](/docs/domain-language) ของ Legal ERP
Platform

## In Scope

- **Client & Matter Management**: บริหารข้อมูล `Client`, `Matter`,
  ประวัติการติดต่อ เอกสารที่เกี่ยวข้อง ทีมผู้รับผิดชอบ และสถานะของงาน
- **Document Automation Management**: สร้าง `Document` จาก `Document Template`,
  auto-fill fields, conditional content, upload, version control, `Audit Log`,
  approval workflow, e-signature, export และ share document
- **Quotation & Service Fee Definition**: สร้าง `Quotation`, กำหนดขอบเขตบริการ,
  pricing model, payment terms, approval workflow, revision history, export,
  print และ dashboard สำหรับติดตาม quotation
- **Calendar & Scheduling**: จัดการนัดหมาย กำหนดเวลาสำคัญ และ reminder ที่ผูกกับ
  `Matter` หรือ `Task`
- **Task & Team Management**: มอบหมาย `Task` ให้ทีม ติดตามความคืบหน้า
  และจัดการผู้รับผิดชอบในแต่ละ `Matter`
- **Billing & Payment Collection**: ติดตามการวางบิล เงื่อนไขการชำระ การรับชำระ
  และสถานะ collection ที่เกี่ยวข้องกับ `Quotation` หรือ `Matter`
- **Finance Foundation**: วางรากฐานข้อมูลการเงิน เช่น chart of accounts,
  receivable, payable, `Invoice`, `Payment`, tax-related setup และ financial
  reporting ที่จำเป็นต่อการต่อยอด module การเงิน
- **Administration & SaaS Management**: จัดการ `Tenant`, `User`, `Role`,
  `Permission`, RBAC, `Audit Log`, security setting, master data และ
  configuration ที่รองรับ multi-tenant

## Out of Scope

รายการต่อไปนี้ยังไม่ถือเป็นขอบเขตหลักของ baseline ปัจจุบัน เว้นแต่จะมี decision
หรือ requirement เพิ่มเติมในภายหลัง:

- Native mobile application
- Production integration กับระบบภายนอก เช่น e-Filing, CIOS, bank, tax system,
  payment gateway, external ERP หรือ external CRM
- Payment gateway workflow แบบเต็มรูปแบบสำหรับรับชำระเงินจริง
- Data migration เต็มรูปแบบจากระบบเดิม ยกเว้นการเตรียม master data import
  ที่ตกลงเพิ่มเติม
- Cloud hosting cost, license cost และ third-party service subscription
- Formal security certification, penetration test หรือ external audit โดยผู้ให้
  บริการภายนอก
- End-user training และ change management สำหรับการ rollout ใช้งานจริง

## Open Questions

- Finance module ใน phase แรกต้องครอบคลุมเพียง foundation และ reporting
  เบื้องต้น หรือรวม accounting workflow แบบเต็มรูปแบบ
- HRM/payroll requirement จะอยู่ใน phase เดียวกับ core legal workflow
  หรือแยกเป็น phase ถัดไป
- Payment collection ต้องจบที่การติดตามสถานะการรับชำระ หรือรวมการเชื่อมต่อ
  payment gateway ด้วย
- External integrations ใดต้องเป็น mock/prototype และ integration
  ใดต้องใช้งานจริงใน production
