# Scope

หน้านี้ตอบ 3 เรื่องหลักของ Legal Practice ERP Platform: ขอบเขตเริ่มต้นรวมอะไร
ยังไม่รวมอะไร และมีคำถามใดที่ต้องตัดสินใจต่อ ก่อนลงรายละเอียด module workflow,
database และ API

## In Scope

- **Client & Matter Management**: บริหารข้อมูลลูกความ แฟ้มงานกฎหมาย
  ประวัติการติดต่อ เอกสารที่เกี่ยวข้อง ทีมผู้รับผิดชอบ สถานะของงาน
  และการบันทึกเวลาทำงานของทนาย (Time Tracking & Timesheets) ทั้งแบบจับเวลา
  และบันทึกย้อนหลัง พร้อมขั้นตอนอนุมัติก่อนเชื่อมเข้าสู่การวางบิล
- **Document Automation Management**: สร้าง Document (เอกสารทางกฎหมาย) จาก
  Document Template (แม่แบบเอกสารทางกฎหมาย), ให้ระบบช่วยเติมข้อมูล,
  เลือกข้อความตามเงื่อนไข, อัปโหลดไฟล์, เก็บเวอร์ชัน,
  บันทึกประวัติการเปลี่ยนแปลง, ส่งอนุมัติ, ลงนามอิเล็กทรอนิกส์, ส่งออก
  และแชร์เอกสาร
- **Quotation & Service Fee Definition**: สร้าง quotation, กำหนดขอบเขตบริการ
  รูปแบบค่าบริการ เงื่อนไขการชำระ ขั้นตอนอนุมัติ ประวัติการแก้ไข การส่งออก
  การพิมพ์ และหน้าสรุปสถานะ quotation
- **Calendar & Scheduling**: จัดการนัดหมาย กำหนดเวลาสำคัญ
  และการแจ้งเตือนที่ผูกกับแฟ้มงานกฎหมายหรือ task
- **Task & Team Management**: มอบหมายงานย่อยให้ทีม ติดตามความคืบหน้า
  และจัดการผู้รับผิดชอบในแต่ละแฟ้มงานกฎหมาย
- **Billing & Payment Collection**: ติดตามการวางบิล เงื่อนไขการชำระ การรับเงิน
  และสถานะการติดตามชำระที่เกี่ยวข้องกับใบเสนอราคาหรือแฟ้มงานกฎหมาย
- **Finance Foundation**: วางข้อมูลตั้งต้นด้านการเงิน เช่น ผังบัญชีเบื้องต้น
  รายการรับเงิน รายการจ่ายเงิน invoice, payment, ข้อมูลภาษี
  และรายงานการเงินเบื้องต้นที่จำเป็นต่อการต่อยอด module การเงิน
- **Human Resource Management**: บันทึกข้อมูลบุคลากร คำนวณเงินเดือน (Payroll)
  ภาษี ประกันสังคม โบนัส ออก payslip ออนไลน์ และระบบลาออนไลน์ (Leave Management)
- **Reports & Analytics**: รายงานและ dashboard สรุปภาพรวมของแต่ละโมดูล เช่น คดี
  ลูกค้า เวลาทำงาน งานค้าง ใบเสนอราคา และการเงิน โดยส่งออกได้อย่างน้อย เป็น
  Excel/PDF/CSV
- **Administration & SaaS Management**: จัดการ tenant, user, role, permission,
  ประวัติการเปลี่ยนแปลง, ความปลอดภัย, master data และค่าตั้งต้นที่รองรับหลาย
  องค์กรในระบบเดียว รวมถึงขั้นตอนสมัครใช้งานสำหรับสำนักงานใหม่ (Tenant
  Onboarding / Self-Service Signup), การสร้าง BU Account พร้อม domain/sub-domain
  ของแต่ละองค์กร, การกำหนดแพ็กเกจการใช้งาน (Package Management)
  ที่ควบคุมว่าโมดูลใดเปิดใช้ได้บ้าง และการควบคุมจำนวน user สูงสุด
  ต่อองค์กรตามแพ็กเกจ (User Limit Control)
