## Purpose

กำหนดวิธีขอ ออกแบบ ตรวจสอบ ทดสอบ เปิดใช้ เฝ้าระวัง เปลี่ยน และยุติ API หรือ
integration โดยรักษา Tenant scope, least privilege, secret security,
compatibility และ idempotency

## Scope

ครอบคลุม Core/Front/Middle/Back Office/Open API, API gateway, documentation,
usage/rate limiting, provider-neutral adapter, inbound/outbound webhook,
email/calendar/e-signature/payment/bank/accounting/HR/court connection,
credential/key/certificate, sandbox/UAT, schema/version, activation, monitoring,
retry/replay, rotation, suspension และ decommission

ไม่ครอบคลุม business approval/transaction ของ Document, Task/Calendar, Billing,
Finance หรือ HR และไม่ถือว่า production provider พร้อมเพียงเพราะ adapter ผ่าน
sandbox ผู้ให้บริการแต่ละรายต้องผ่าน security/consent/contract/release gate

| Role                  | Responsibility                                                                             |
| --------------------- | ------------------------------------------------------------------------------------------ |
| Integration Owner     | รับผิดชอบ connection purpose, data flow, lifecycle, SLA และ provider relationship          |
| Domain Owner          | ยืนยัน business fields/events, reconciliation และ fallback ของ domain                      |
| Technical Owner       | ออกแบบ API/adapter/schema, test, observability, retry และ compatibility                     |
| Security Reviewer     | ตรวจ authentication, secret, scope, encryption, provider/data risk                         |
| Manager A1            | อนุมัติ business purpose, data flow และ production activation                              |
| Admin A2              | อนุมัติ privileged/cross-domain/external connection และแยกจาก executor                     |
| Admin/System          | provision gateway/credential/config หลัง approval และบันทึก immutable events               |
| Operations Owner      | monitor health/usage/error และส่ง incident ไป SOP-OPS-001/CTRL-EXC-001                     |
| Data/Privacy Owner    | ยืนยัน classification, lawful purpose, retention และ external recipient เมื่อมี sensitive data |

## Approved Status Baseline

### Integration Connection

Draft, Pending Review, Approved, Sandbox, Active, Suspended, Failed และ Retired

### Credential

Pending, Active, Rotating, Revoked และ Expired

- Approved ยังห้ามส่ง production data จน Active
- Sandbox ใช้ synthetic/masked data ตาม classification
- Suspended block traffic แต่คง configuration/history
- Failed ต้องมี owner/fallback และห้ามเปลี่ยนเป็น Active โดยแก้สถานะ
- Retired/Revoked ห้ามกลับใช้ credential/config เดิม

## Required Information

- Integration/Connection ID, type, owner, Tenant/environment, provider/endpoint
  และ business purpose
- source/destination system, direction, events/fields, classification, volume,
  frequency, retention และ data residency
- API/adapter/schema/event version, compatibility policy, documentation,
  timeout/rate limit/retry/backoff/idempotency/replay rules
- authentication method, credential owner, secret/certificate reference,
  scopes, expiry/rotation date และ network/allowlist policy
- sandbox/UAT fixtures, contract/security/privacy/consent result, control totals,
  negative tests และ fallback/manual path
- requester, Domain/Integration/Technical/Security/Data owners, A1/A2, executor,
  change/release/incident IDs และ timestamps

## Procedure

### 1. Submit Integration Request

**Responsible:** Requester, Integration Owner, Domain Owner

1. ระบุ purpose, systems, direction, events/fields, Tenant/environment และ owner
2. classify data, recipients, residency, frequency/volume และ retention
3. ระบุ production provider หรือ provider-neutral stub/adapter พร้อม fallback
4. ตรวจว่าความต้องการไม่ซ้ำ connection/API ที่มีอยู่
5. สร้าง Connection ID และคง Draft โดยยังไม่ออก production credential

**Expected result:** request มี business/data boundary และ ownership ครบ

### 2. Design API, Schema and Reliability Controls

**Responsible:** Technical Owner, Domain Owner

1. กำหนด endpoint/event, method, schema/version และ compatibility/deprecation
2. ระบุ authentication, scope, timeout, rate limit, payload/file size และ clock
3. กำหนด idempotency, ordering, retry/backoff, dead-letter/replay และ duplicate rule
4. ระบุ source/target control totals, acknowledgment และ reconciliation
5. จัดทำ API documentation/example โดยไม่มี production secret/personal data

**Expected result:** contract ทดสอบได้และไม่ฝาก correctness ไว้ที่ manual guess

### 3. Perform Security, Privacy and Provider Review

**Responsible:** Security Reviewer, Data/Privacy Owner, Integration Owner

1. ตรวจ provider identity/contract, authentication, encryption, logging, breach/
   subprocessor/data-residency และ deletion/exit terms ตาม scope
2. ใช้ least-privilege credential scope และแยก sandbox/production/Tenant
3. เก็บ raw secret ใน approved secret store; evidence เก็บเฉพาะ reference/version
4. ประเมิน sensitive data, lawful purpose/consent, minimization และ retention
5. risk/exception ต้องมี owner/expiry/authority ห้ามถือ silence เป็น approval

**Expected result:** Pending Review ผ่าน security/privacy/provider gate

### 4. Provision Sandbox and Test

**Responsible:** Admin/System, Technical Owner, Domain Owner

1. provision sandbox endpoint/credential/config ด้วย approved request
2. ใช้ synthetic/masked fixture และ block unintended production delivery
3. ทดสอบ happy path, authorization, cross-Tenant denial, invalid schema,
   timeout/rate limit, duplicate/out-of-order/retry และ provider failure
4. reconcile input/accepted/rejected/processed/control totals และ audit events
5. Domain Owner ยืนยัน business result/fallback; failed test ต้องแก้และ rerun

**Expected result:** Sandbox/UAT evidence ครบโดยไม่ส่ง production data

### 5. Approve and Activate Production

**Responsible:** Manager A1, Admin A2, Admin/System

1. submit exact connection/config/schema/credential version พร้อม UAT/release
2. Manager A1 อนุมัติ purpose/data flow; external/privileged/cross-domain ใช้ A2
3. requester/approver/executor ต้องแยกตาม risk
4. provision production credential จาก secret store และ activate ตาม change window
5. smoke test แบบ controlled, verify monitoring/reconciliation/fallback แล้ว Active

**Expected result:** production traffic เริ่มหลัง approval และ readiness ครบ

### 6. Operate, Monitor and Reconcile

**Responsible:** Operations Owner, Integration Owner, Domain Owner

1. monitor availability/latency/error/rate-limit/queue/retry/credential expiry
2. correlate request/event/idempotency/provider IDs โดยไม่ log sensitive payload
3. retry/replay เฉพาะตาม policy; duplicate callback คืนผลเดิมไม่ทำธุรกรรมซ้ำ
4. reconcile scheduled totals/status/acknowledgment กับ source และ target
5. incident ส่ง SOP-OPS-001/CTRL-EXC-001; business correction ใช้ SOP domain

**Expected result:** connection Active มี health, totals และ exception owner

### 7. Change Schema, Scope, Provider or Credential

**Responsible:** Integration/Technical/Security/Domain Owners, Admin/System

1. เปิด change ระบุ before/after, compatibility, consumers, migration/fallback
2. scope/data/provider/authentication change ต้อง security/privacy review ใหม่
3. schema breaking change ใช้ version ใหม่, overlap/deprecation notice และ UAT
4. rotate credential ก่อน expiry โดยทดสอบ new version แล้ว revoke old version
5. activate ตาม approval; rollback ใช้ approved previous config ไม่แก้ event เดิม

**Expected result:** change ไม่ตัด consumer เงียบและไม่มี credential ค้าง

### 8. Suspend or Retire Connection

**Responsible:** Integration Owner, Manager A1/Admin A2, Admin/System

1. suspend เร่งด่วนเมื่อ compromise/data risk/duplicate transaction โดย preserve log
2. retire plan ระบุ consumers, in-flight items, final reconciliation/export/retention
3. แจ้ง Domain Owner/provider และย้ายไป replacement/manual fallback
4. revoke credential, webhook, allowlist, scheduled job และ access path
5. เปลี่ยน Retired หลัง traffic/queue/totals/secret/log handoff reconcile

**Expected result:** connection หยุดอย่างตรวจสอบได้และไม่เหลือ secret/traffic

## Control Points

| Control                   | Requirement                                                              | Evidence                                               |
| ------------------------- | ------------------------------------------------------------------------ | ------------------------------------------------------ |
| Ownership/data boundary   | owner, purpose, Tenant, source/target, fields/classification ชัด          | request and data-flow/version snapshot                 |
| Contract/version          | documented schema/event/API version และ compatibility/deprecation         | contract/docs and consumer test result                 |
| Security/privacy gate     | provider/auth/scope/secret/data risk ผ่านก่อน production                  | review decisions and secret reference                 |
| Sandbox/UAT isolation     | synthetic/masked fixture และ block production delivery                    | environment/fixture/access result                      |
| Gateway/reliability       | authentication, rate limit, timeout, retry/idempotency/replay enforced    | gateway/config/version and negative-test evidence      |
| Reconciliation            | source/target/input/accepted/rejected/processed totals ตรง                 | batch/event control totals and variance               |
| Credential lifecycle      | separated environment/Tenant, expiry/rotation/revoke และ no raw secret log | credential reference and rotation/revocation events   |
| Suspension/retirement     | preserve evidence, revoke all paths, settle in-flight and fallback        | before/after, revoke, final totals and owner acceptance |

## Exceptions

- **Provider/security review ไม่ผ่าน:** คง Pending Review ใช้ stub/manual path;
  ห้ามออก production credential
- **Schema/version ไม่เข้ากัน:** reject/quarantine payload และแจ้ง owner; ห้าม
  silently drop/rename field
- **Rate limit/timeout:** backoff/queue/fallback ตาม policy ห้ามยิง retry ไม่จำกัด
- **Duplicate/out-of-order:** ใช้ idempotency/sequence rule และ reconcile ห้าม
  post/send business action ซ้ำ
- **Credential leak/compromise:** suspend/revoke/rotate ทันที preserve evidence
  และเปิด security incident
- **Provider outage:** คง local source state, retry/fallback และแจ้ง SLA; ห้าม mark
  Delivered/Posted จาก request ที่ยังไม่ acknowledge
- **Reconciliation mismatch:** stop affected replay/close เปิด exception และให้
  Domain Owner ตัดสิน correction
- **Contract/consent/retention หมดอายุ:** suspend production data flow จน renew
