## Purpose

กำหนดวิธีสร้างและดูแลเอกสารทางกฎหมายให้ใช้ template/ข้อมูลที่ถูกต้อง มี version
และ approval ที่ตรวจย้อนหลังได้ พร้อมควบคุม signature, sharing, access และ
retention

## Scope

ครอบคลุม Document List/Search, template generation, auto-fill, conditional
content, upload, metadata, version, lock, review, approval, e-signature, export,
share, expiry และ archive

## Roles

| Role      | Responsibility                                                                                    |
| --------- | ------------------------------------------------------------------------------------------------- |
| Lawyer    | สร้าง แก้ review และรับผิดชอบความถูกต้องทางกฎหมาย                                                 |
| Assistant | อัปโหลด จัดหมวดหมู่ metadata และเตรียม draft                                                      |
| Manager   | อนุมัติ final document และ external release เป็น A1                                               |
| Admin     | ดูแล template/configuration/RBAC/signature profile และอนุมัติ Restricted/Privileged share เป็น A2 |

Admin ห้ามอนุมัติเนื้อหาเอกสารทางกฎหมายหรือลงนามแทนเจ้าของลายเซ็น Admin A2
ตรวจได้เฉพาะ metadata, classification, recipient, purpose, expiry และ approval
evidence และต้องเป็นคนละบัญชีกับ Admin ผู้ execute share เดียวกัน

## Document Status

| Status            | Meaning                                           |
| :-----------------: | ------------------------------------------------- |
| Draft             | กำลังจัดทำและแก้ไขได้ตาม permission               |
| In Review         | ส่งให้ reviewer และ version ถูกตรึงระหว่าง review |
| Changes Requested | Reviewer ส่งกลับพร้อมเหตุผล                       |
| Approved          | ผ่าน final approval และพร้อม external release     |
| Pending Signature | ส่งคำขอลงนามแล้ว                                  |
| Signed            | ลงนามครบตาม signing order                         |
| Expired           | พ้น validity/expiry date                          |
| Superseded        | มี approved/signed version ใหม่แทน                |
| Cancelled         | ยกเลิกก่อน final use พร้อมเหตุผล                  |

## Required Information

- Document ID, Tenant ID, title, document type และ confidentiality
- Matter ID และ Client ID เมื่อเกี่ยวข้อง
- Template ID/version หรือ upload source
- Category, tags, owner และ reviewers
- Version number, checksum, created/updated by และ timestamps
- Approval route และ signing route เมื่อเกี่ยวข้อง
- Expiry date, retention policy/version และ Legal Hold status

## Procedure

### 1. Create or Upload a Document

**Responsible:** Lawyer หรือ Assistant

1. ค้นหาเอกสารเดิมตาม title, tag, Matter, Client และ date
2. เลือก active Template version หรือ upload file type/size ที่อนุญาต
3. Generate จาก template โดยระบุ Matter ID และ Client ID
4. ตรวจ auto-fill mapping และ conditional rule result
5. ระบุ category, tags, owner, confidentiality และ retention
6. ตรวจ malware/file integrity และสร้าง checksum
7. บันทึกเป็น Draft และตรวจ preview

**Expected result:** Draft document มี metadata, linkage และไฟล์ที่ผ่าน
validation

### 2. Edit and Version the Document

**Responsible:** Lawyer หรือ Assistant ตาม permission

1. ตรวจ lock และเปิด version ล่าสุดที่แก้ได้
2. แก้ content/metadata โดยไม่เขียนทับ version เดิม
3. สร้าง version ใหม่ทุกครั้งที่บันทึก substantive change
4. บันทึก change summary, editor และ timestamp
5. Lock เมื่อส่ง review หรือป้องกัน concurrent edit
6. ปลด lock ได้โดย owner หรือ Manager พร้อม reason

**Expected result:** Version history ครบและย้อนดู version เดิมได้

### 3. Review and Approve

**Responsible:** Lawyer/Assistant (Preparer), Manager (Approver)

1. ตรวจ required fields, links, citations, attachments และ final preview
2. ส่ง version เป็น In Review และ assign approver ตาม document type
3. Approver ตรวจเนื้อหา scope, confidentiality และ external-use readiness
4. Changes Requested ต้องระบุ reason และสร้าง revision ใหม่เมื่อแก้
5. Approve เฉพาะ version ที่ review แล้วและผู้เตรียมไม่ใช่ผู้อนุมัติ
6. เก็บ approver, decision, version, reason และ approved at

**Expected result:** Approved version ถูกตรึงและพร้อมลงนามหรือเผยแพร่

### 4. Sign the Document

**Responsible:** Document Owner, Authorized Signer

1. ตรวจว่า version เป็น Approved และ signer/email/signing order ถูกต้อง
2. สร้าง signing request ที่มี expiry และ access token จำกัดขอบเขต
3. Signer ยืนยันตัวตนและลงนามในตำแหน่งที่กำหนด
4. บันทึก signer, method, consent, timestamp และ signature evidence
5. เมื่อครบทุก signer ให้เปลี่ยนเป็น Signed และสร้าง final checksum
6. ห้ามแทนที่ signed file; การแก้ต้องสร้าง version/request ใหม่

**Expected result:** Signed document และ signature evidence ตรวจสอบย้อนหลังได้

### 5. Export or Share

**Responsible:** Document Owner ตาม permission

1. ตรวจ status, recipient, purpose, format และ confidentiality
2. External share ใช้เฉพาะ Approved/Signed version เว้นแต่มี watermark Draft และ
   Manager อนุมัติ
3. Restricted/Privileged external share ต้องมี Manager และ Admin อนุมัติ
4. ใช้ expiring link, recipient authentication และ download limit ตาม policy
5. บันทึก exported/shared version, recipient, channel, expiry และ actor

**Expected result:** ผู้รับเข้าถึงเฉพาะ version และช่วงเวลาที่อนุมัติ

### 6. Expire, Supersede or Archive

**Responsible:** Document Owner, Manager

1. ระบบแจ้ง owner ก่อน expiry ตาม Tenant Policy
2. ระบุ Expired หรือ Superseded พร้อม replacement Document ID เมื่อมี
3. ถอน active share link ที่ไม่ควรใช้ต่อ
4. Archive โดยรักษา version, signature, access history และ checksum
5. Disposal ใช้ SOP-AUD-001 และ Legal Hold

**Expected result:** เอกสารพ้นการใช้งานปกติแต่ประวัติและหลักฐานยังครบ

## Control Points

| Control              | Requirement                                             | Evidence                                   |
| -------------------- | ------------------------------------------------------- | ------------------------------------------ |
| Active template      | Generate ได้จาก active Template version เท่านั้น        | Template ID/version และ validation result  |
| Version immutability | ห้ามเขียนทับ approved/signed หรือ version เดิม          | Version chain, checksum และ change summary |
| Final approval       | Preparer และ approver ต้องเป็นคนละ User                 | Approval record และ SoD result             |
| Access control       | List/search/preview/download กรองตาม Matter/Tenant/RBAC | Permission decision และ access event       |
| External release     | ตรวจ status, recipient, purpose และ confidentiality     | Share/export approval และ delivery event   |
| Signature integrity  | Signer, consent, order และ signed checksum ต้องครบ      | Signature evidence package                 |
| Encryption           | File เข้ารหัส at rest/in transit และ backup ตาม policy  | Encryption/backup metadata                 |
| Retention            | Version และ evidence ใช้ระยะเวลาที่ยาวที่สุด            | Policy version, Legal Hold และ eligibility |

## Exceptions

- **Template mapping ไม่ครบหรือ rule ขัดกัน:** หยุด generate และแจ้ง Template
  Owner/Admin
- **File ไม่ผ่าน validation:** ห้ามจัดเก็บเป็น usable document
- **ต้องแก้ Approved/Signed document:** สร้าง version ใหม่และ re-approve/re-sign
- **Reviewer ไม่อยู่:** ใช้ Alternate Approver ที่มี authority เทียบเท่า
- **ส่งผิดผู้รับ:** ถอน link ทันที บันทึก incident และแจ้ง Admin
- **มี Legal Hold:** ห้าม disposal หรือทำลาย version/signature evidence

## Completion Checklist

- [ ] Template/upload ผ่าน validation และ malware/integrity check
- [ ] Matter/Client, category, tags และ confidentiality ถูกต้อง
- [ ] Version/checksum/change summary ถูกบันทึก
- [ ] Preparer และ approver แยกกัน
- [ ] Approved/Signed version ถูกตรึง
- [ ] Signature evidence ครบเมื่อเกี่ยวข้อง
- [ ] Export/share มี recipient, purpose, expiry และ audit event
- [ ] Retention Policy และ Legal Hold ถูกผูก
