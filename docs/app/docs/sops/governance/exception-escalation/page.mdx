# Exception & Escalation Matrix

| Document Control        | Value                           |
| ----------------------- | ------------------------------- |
| Document ID             | CTRL-EXC-001                    |
| Status                  | Approved                        |
| Version                 | 1.0                             |
| Process Owner           | Manager                         |
| Approver                | Manager + Data/Compliance Owner |
| Effective Date          | 14 July 2026                    |
| Last Requirement Review | 14 July 2026                    |

## Purpose

กำหนดระดับความรุนแรง ผู้รับผิดชอบ SLA การแจ้ง และอำนาจตัดสินใจเมื่อ SOP ทำงาน
ไม่ได้ ผลลัพธ์ผิดปกติ หรือ control/evidence ไม่ครบ

## Severity and Response SLA

| Severity    | Criteria                                                                                                         | Acknowledge      | Contain         | Restore Target   | Resolution / Approved Plan | Update Cadence |
| ----------- | ---------------------------------------------------------------------------------------------------------------- | ---------------- | --------------- | ---------------- | -------------------------- | -------------- |
| P1 Critical | Data breach/cross-Tenant access, unauthorized payment, legal deadline at immediate risk, audit integrity failure | 15 min           | 1 hour          | 4 hours          | 24 hours                   | Every 1 hour   |
| P2 High     | Core workflow blocked, privileged access anomaly, material billing/report error, SLA breach with client impact   | 1 hour           | 4 hours         | 24 hours         | 72 hours                   | Every 4 hours  |
| P3 Medium   | Single record/user affected, recoverable sync/version/validation issue, no immediate legal/financial impact      | 4 business hours | 1 business day  | 3 business days  | 10 business days           | Daily          |
| P4 Low      | Cosmetic/documentation issue or low-risk workaround available                                                    | 1 business day   | 5 business days | 10 business days | 30 calendar days           | Weekly/closure |

SLA นับต่อเนื่องสำหรับ P1/P2 และนับเวลาทำการสำหรับ P3/P4 ห้าม auto-approve
หรือปิด exception อัตโนมัติเมื่อ SLA เกิน

## Exception Status

Open, Contained, In Remediation, Resolved, Closed และ Cancelled โดย Resolved
หมายถึง correction ทำงานแล้ว ส่วน Closed ใช้ได้เมื่อ closure evidence/sign-off
ครบ

## Escalation Matrix

| Domain / Trigger                         | First Owner           | Decision Authority                                         | Mandatory Escalation / Evidence                  |
| ---------------------------------------- | --------------------- | ---------------------------------------------------------- | ------------------------------------------------ |
| Client duplicate/merge uncertainty       | Manager               | Manager + Data Owner when material                         | Candidate score, dependency and decision reason  |
| Conflict/Risk/Prohibited finding         | Responsible Lawyer    | Manager; Executive A2 for High; no override for Prohibited | Assessment and decision event                    |
| Matter deadline/status/closure blocked   | Manager               | Manager/Executive per DEC-MAT-012                          | Deadline impact, transition request and notice   |
| Wrong document version/signature/share   | Document Owner        | Manager + Data/Compliance if external/sensitive            | Revoke, recipients, checksum and incident        |
| Calendar conflict/external sync failure  | Task/Calendar Owner   | Manager for deadline impact                                | Conflict, retries, manual confirmation           |
| Quotation pricing/approval inconsistency | Finance               | Manager                                                    | Version, price source, discount and client state |
| Invoice/payment/refund discrepancy       | Finance               | Manager; Data/Compliance for suspected fraud               | Reconciliation, before/after and posting IDs     |
| Unauthorized/privileged access anomaly   | Admin                 | Data/Compliance Owner + Manager                            | Lock/revoke events, scope and session activity   |
| Report mismatch/sensitive export         | Report Owner          | Manager + Data/Compliance when exposed                     | Filter snapshot, rows, recipient and revocation  |
| Legal Hold/disposal conflict             | Data/Compliance Owner | Manager A1 + Data/Compliance A2                            | Hold search, population and disposal block       |
| Audit event integrity/correction failure | Data/Compliance Owner | Manager + Data/Compliance Owner                            | Integrity result, original/correction reference  |

## Immediate Actions

1. หยุด action ที่ทำให้ผลกระทบขยาย เช่น share, posting, export, disposal หรือ
   access
2. รักษาหลักฐาน ห้ามลบ/แก้ event ต้นฉบับ และเปิด FRM-EXC-001
3. ประเมิน Tenant, Client/Matter, record, user, amount, deadline และ data
   exposure
4. กำหนด severity จากผลกระทบสูงสุด ไม่ลดระดับเพียงเพราะมี workaround
5. แจ้ง First Owner และ Decision Authority ตาม SLA
6. ใช้ controlled workaround เฉพาะเมื่อ owner/authority อนุมัติและมี expiry

## FRM-EXC-001 Required Fields

- Exception ID, detected at/by, affected Tenant and related record IDs
- SOP/control/requirement, failed step and expected/actual result
- Severity, impact dimensions and deadline/client/regulatory exposure
- Containment, workaround, owner, due time and expiry
- Decision Authority, approval/rejection and communication log
- Root cause, corrective/preventive action, UAT/regression reference
- Closure evidence, residual risk, closed by/at and follow-up review date

## Escalation Rules

- P1/P2 ต้องแจ้ง Manager และ Data/Compliance Owner แม้ domain owner แก้ได้แล้ว
- Data breach/cross-Tenant access และ disposal under Legal Hold เป็น P1 เสมอ
- Suspected fraud ที่ยังไม่ยืนยันต้องไม่ต่ำกว่า P2 และยกระดับเป็น P1 เมื่อพบ
  unauthorized payment/data exposure หรือ material integrity impact
- หาก owner ไม่ acknowledge ภายใน SLA ให้ escalate ไป Decision Authority ทันที
- หากพลาด Contain target ให้แจ้ง Manager + Data/Compliance Owner และทบทวน
  severity
- หากพลาด Restore target ให้ escalate ไป Executive Approver เพื่ออนุมัติ
  recovery path, client/regulatory communication และ resource เพิ่มเติม
- หากพลาด Resolution/Approved Plan target ต้องมี documented risk acceptance,
  revised due time และ Executive Approver; ห้ามเลื่อนเงียบหรือปิด exception
- หากพลาด Update Cadence ให้ escalation system แจ้ง owner manager และ Decision
  Authority โดยไม่ reset SLA clock
- หาก workaround หมดอายุหรือทำซ้ำเกินสองครั้ง ให้ยกระดับหนึ่งระดับและเปิด
  corrective action
- Exception ที่กระทบ contract/law/deadline ต้องให้ Legal/Compliance
  พิจารณาการแจ้งภายนอก
- Self-approval ใช้ไม่ได้; หาก approver เป็นผู้เกี่ยวข้องต้องใช้ alternate
  approver

## Closure Criteria

- Containment และ permanent correction ทำงานแล้ว
- Data/financial/status reconciliation ครบและไม่มี orphaned record
- ผู้ได้รับผลกระทบได้รับแจ้งตาม decision
- Evidence package ครบและลิงก์กลับไป requirement/UAT ได้
- Root cause และ preventive action มี owner/due date
- P1/P2 ผ่าน post-incident review และ Data/Compliance sign-off
