## Purpose

กำหนดข้อมูลขั้นต่ำของแบบฟอร์ม checklist และ evidence package ที่ใช้กับ SOP
เพื่อให้ระบบและผู้ปฏิบัติงานบันทึกข้อมูลในรูปแบบเดียวกันและตรวจย้อนหลังได้

## Common Record Header

ทุกแบบฟอร์มต้องมีข้อมูลต่อไปนี้:

| Field                  | Requirement                                                    |
| ---------------------- | -------------------------------------------------------------- |
| Record ID              | ID ไม่ซ้ำและสร้างโดยระบบ                                       |
| Form ID / Version      | Template ID และ version ที่ใช้                                 |
| Tenant ID              | Tenant เจ้าของข้อมูล                                           |
| Related Record         | Client, Matter, Document, Task, Billing หรือ User ID           |
| Request Type           | ประเภท action ที่ร้องขอ                                        |
| Requester / Owner      | ผู้ขอและผู้รับผิดชอบ                                           |
| Reason / Purpose       | เหตุผลเชิงธุรกิจหรือ control objective                         |
| Created / Effective    | วันที่สร้างและวันที่มีผล                                       |
| Status                 | Draft, Submitted, Approved, Rejected, Cancelled หรือ Completed |
| Approvals              | approver, decision, timestamp และ comment                      |
| Attachments / Evidence | reference, checksum/version และ classification                 |
| Audit Reference        | immutable event หรือ workflow instance ID                      |

## Evidence Quality Rules

- Evidence ต้องระบุ actor, action, object, timestamp, outcome และ Tenant
- Snapshot ต้องมี version/checksum หรือ immutable reference ไม่ใช้ screenshot
  อย่างเดียวเมื่อมี structured event
- Approval comment ต้องอธิบาย exception/override; approval
  เปล่าหรือย้อนหลังโดยไม่มีเหตุผลใช้ไม่ได้
- Evidence classification และ retention ต้องเท่ากับหรือเข้มกว่าข้อมูลต้นทาง
- การแก้ evidence ใช้ correction event เชื่อม original ห้ามเขียนทับ
- Attachment ที่เข้ารหัสหรือเปิดไม่ได้ถือว่า evidence ไม่ครบ
