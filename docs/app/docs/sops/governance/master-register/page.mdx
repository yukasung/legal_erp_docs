## Purpose

เป็นทะเบียนกลางสำหรับควบคุมรหัส ชื่อ เจ้าของ ผู้อนุมัติ เวอร์ชัน สถานะ
วันที่มีผล URL และรอบทบทวนของ SOP, control, standard, template และ assurance
record ทั้งหมด โดย form template รายฉบับมอบอำนาจให้ Template Catalog ใน
TPL-SOP-001 ควบคุม

## Status Rules

| Status     | Use                                                       |
| ---------- | --------------------------------------------------------- |
| Draft      | อยู่ระหว่างจัดทำ ห้ามใช้เป็นหลักฐานการปฏิบัติงาน          |
| In Review  | ส่งให้ Process Owner/Approver ตรวจ แต่ยังไม่มีผลบังคับใช้ |
| Approved   | ได้รับอนุมัติและใช้ตาม Effective Date                     |
| Superseded | มีฉบับใหม่แทน ต้องเก็บฉบับเดิมและลิงก์ไปยังฉบับใหม่       |
| Retired    | ยกเลิกโดยไม่มีฉบับแทน ต้องระบุเหตุผลและวันสิ้นสุด         |

ห้ามแก้เนื้อหาเอกสาร Approved โดยไม่เพิ่ม version และสร้าง change record

## Review Triggers

ทบทวนก่อนถึงรอบตามตารางเมื่อเกิดเหตุใดเหตุหนึ่ง:

- Requirement, law, retention obligation หรือ client contract เปลี่ยน
- Workflow, status, role, permission หรือ approval authority เปลี่ยน
- Release มีผลต่อ control หรือ evidence ที่ SOP กำหนด
- Incident, audit finding, UAT failure หรือ exception ซ้ำแสดงว่า control ไม่พอ
- Process Owner, Approver หรือระบบหลักเปลี่ยน

## Change Control

1. ผู้ขอเปลี่ยนระบุ document ID, เหตุผล, requirement/control ที่กระทบ
   และวันที่ต้องการ
2. Process Owner ประเมินผลต่อ SOP, template, permission, traceability และ UAT
3. Minor editorial change เพิ่ม patch version; control/behavior change เพิ่ม
   minor หรือ major version
4. Approver ต้องอนุมัติก่อน Effective Date และห้ามผู้จัดทำอนุมัติรายการของตนเอง
5. อัปเดตทะเบียน ลิงก์ เอกสารที่เกี่ยวข้อง และเก็บฉบับเดิมเป็น Superseded
6. แจ้งผู้ใช้งานที่ได้รับผลกระทบและบันทึก acknowledgement เมื่อเป็น material
   change

## Register Controls

- Admin ตรวจทะเบียนทุกไตรมาสและติดตาม overdue review
- ทุก ID ต้องไม่ซ้ำและ URL ต้องเปิดได้
- ทุก Approved document ต้องมี owner, approver, version และ effective date
- เอกสารที่ถูกแทนต้องไม่หายจาก audit history
- Register correction ต้องเก็บ before/after, reason, actor และ timestamp
