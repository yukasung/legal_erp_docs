| Document Control        | Value        |
| ----------------------- | ------------ |
| Document ID             | STD-SOP-001  |
| Status                  | Approved     |
| Version                 | 1.0          |
| Process Owner           | Manager      |
| Approver                | Manager      |
| Effective Date          | 14 July 2026 |
| Last Requirement Review | 14 July 2026 |

## Purpose

กำหนดคำศัพท์ ชื่อสถานะ รูปแบบวันที่ และกติกาลิงก์มาตรฐานสำหรับ SOP, หน้าจอ,
รายงาน, event และ test case เพื่อป้องกันความหมายเดียวกันแต่ใช้คนละคำ

## Decision and Transition Language

- ใช้ approve/reject กับ decision และใช้ transition กับการเปลี่ยน status
- ใช้ deactivate/reactivate กับ Client/User; ใช้ close/reopen กับ Matter
- ใช้ void กับ financial document ที่ยกเลิกผลทางบัญชี และใช้ reverse
  กับรายการชดเชยทางการเงิน
- ใช้ supersede เมื่อมี version ใหม่แทน; archive หมายถึงย้ายสู่ controlled
  storage ไม่ใช่ disposal
- ใช้ delete เฉพาะข้อมูลชั่วคราวที่ policy อนุญาต ข้อมูลสำคัญใช้ cancel,
  deactivate, void หรือ supersede

## Date, Time and Number Standard

- Controlled documents ใช้วันที่แบบ 14 July 2026 เพื่ออ่านตรงกันข้าม locale
- System timestamp และ integration ใช้ ISO 8601 พร้อม timezone เช่น
  2026-07-14T09:00:00+07:00
- Calendar/deadline ต้องเก็บ timezone และแสดง locale ของผู้ใช้โดยไม่เปลี่ยน
  instant
- จำนวนเงินต้องมี currency; rate/discount/tax ต้องเก็บหน่วยและ precision
- Report ต้องมี as-of timestamp, filters และ data source/version

## Link and Naming Rules

- ลิงก์ภายในใช้ canonical route ที่ขึ้นต้นด้วย /docs และไม่ใส่นามสกุลไฟล์
- Related Documents ใช้ชื่อ H1 จริงของหน้าปลายทาง
- เมื่อเปลี่ยนชื่อ/route ต้องทำ redirect และตรวจ broken links ก่อน release
- เมนูใช้ชื่อสั้นที่ยังคงความหมายเดียวกับ H1 และจัด Governance ไว้หลัง
  operational SOP
- ห้ามสร้างหน้าใหม่ชื่อใกล้เคียงเพื่อแก้คำผิด ให้แก้ canonical page และ link
  เดิม

- [ ] Related Documents และ menu link เปิดได้
