## Coverage Rules

- Requirement ใหม่ต้องได้ ID ก่อน development และห้าม reuse ID ที่ยกเลิก
- ทุก Requirement ต้องมี owner, source, implementation/control และ UAT
  อย่างน้อยหนึ่งรายการ
- หากยังไม่มี Decision ให้ระบุ policy owner และสถานะ Draft ใน change request
  ห้ามสมมติว่าอนุมัติแล้ว
- UAT ที่ Failed หรือ Blocked ทำให้ requirement นั้นยังไม่พร้อม release
- Requirement ที่ยกเลิกต้องระบุเหตุผล approver และ release ที่ยกเลิก
  โดยคงประวัติเดิม

## Release Review

1. Product/Business Owner ยืนยัน scope และ Requirement ID ของ release
2. Process Owner ยืนยัน SOP/control และ evidence ที่ได้รับผลกระทบ
3. QA Lead map test case และผลทดสอบกลับมายัง Requirement ID
4. Data/Compliance Owner ตรวจ security, retention และ audit coverage
5. Release Approver ยืนยันว่า Critical/High requirement ผ่านหรือมี accepted
   exception
