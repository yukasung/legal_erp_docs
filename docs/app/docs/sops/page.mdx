# SOPs

Standard Operating Procedure (SOP) ในส่วนนี้อธิบายวิธีปฏิบัติงานของผู้ใช้
ตั้งแต่เริ่มต้นจนได้ผลลัพธ์ที่ตรวจสอบย้อนหลังได้ โดยอ้างอิง capability และ
workflow จาก requirement ของ Legal ERP Platform

SOP แตกต่างจากหน้า Modules ดังนี้:

| Documentation | Main Question    | Content                                            |
| ------------- | ---------------- | -------------------------------------------------- |
| Modules       | ระบบรองรับอะไร   | capability, data และ business rule ของ module      |
| SOPs          | ผู้ใช้ต้องทำอะไร | ผู้รับผิดชอบ ลำดับงาน จุดควบคุม ผลลัพธ์ และหลักฐาน |

## Status Definitions

| Status     | Meaning                                                   |
| ---------- | --------------------------------------------------------- |
| Draft      | อยู่ระหว่างจัดทำ ห้ามใช้เป็นหลักฐานการปฏิบัติงาน          |
| In Review  | ส่งให้ Process Owner/Approver ตรวจ แต่ยังไม่มีผลบังคับใช้ |
| Approved   | ได้รับอนุมัติและใช้ตาม Effective Date                     |
| Superseded | มีฉบับใหม่แทน โดยยังเก็บฉบับเดิมใน audit history          |
| Retired    | ยกเลิกโดยไม่มีฉบับแทน พร้อมเหตุผลและวันสิ้นสุด            |

## Review Rules

- ทุก SOP ต้องระบุ source requirement ที่ตรวจสอบย้อนกลับได้
- ข้อมูลที่ source ไม่ได้กำหนดต้องมี Decision/Policy Owner และ approval ก่อนใช้
- การเปลี่ยน role, approval, permission หรือหลักฐานที่ต้องเก็บ ต้องผ่าน Process
  Owner ก่อนเปลี่ยนสถานะเป็น Approved
- ทบทวน SOP เมื่อ requirement, workflow, permission หรือ policy ขององค์กรเปลี่ยน
- ใช้ Master Register, Traceability Matrix และ UAT result เป็น release evidence
