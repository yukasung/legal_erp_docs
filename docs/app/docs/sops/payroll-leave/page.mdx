## Purpose

กำหนดวิธีจัดทำและอนุมัติรอบเงินเดือน คำนวณภาษี ประกันสังคม และกองทุน ออก payslip
ออนไลน์ ส่งออกข้อมูลให้ทีมบัญชี และจัดการระบบลาออนไลน์ของบุคลากร
โดยแยกข้อมูลเงินเดือนรายบุคคลออกจากข้อมูล billing ของลูกค้าที่ Finance ดูแล

## Scope

ครอบคลุม Payroll Run (base pay, OT, เบี้ยขยัน, โบนัส, ภาษีหัก ณ ที่จ่าย,
ประกันสังคม, กองทุนสำรองเลี้ยงชีพ), Payslip generation, Export ข้อมูลเงินเดือน
ให้ทีมบัญชี และ Leave Request/Approval/Balance tracking ไม่ครอบคลุมข้อมูล
Matter, Client billing หรือ legal content ใด ๆ

## Roles

| Role                             | Responsibility                                                                                                           |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| HR                               | จัดทำรอบเงินเดือน คำนวณภาษี/ประกันสังคม/กองทุน ออก payslip และดูแล leave request                                         |
| Manager                          | อนุมัติรอบเงินเดือน (A1) และคำขอลาของทีมตนเอง                                                                            |
| Admin                            | อนุมัติ A2 สำหรับ payroll run แบบ privileged/ทั้งองค์กร (คนละบัญชีจาก HR) และตั้งค่า master data ภาษี/ประกันสังคม/กองทุน |
| ผู้ใช้งานทุก role (self-service) | ยื่นคำขอลาและดู payslip ของตนเองผ่านบัญชีที่มีอยู่ ไม่ใช่ role แยก                                                       |

Finance ไม่มีสิทธิ์เข้าถึงข้อมูลเงินเดือนรายบุคคลหรือ payroll run ตาม
[Role & Permission Alignment](/docs/sops/governance/role-permission-alignment)

## Approved Status Baseline

### Payroll Run

Draft, Calculated, Pending Approval, Approved, Posted และ Cancelled

### Leave Request

Draft, Submitted, Approved, Rejected, Cancelled และ Taken

Draft ห้ามส่งอนุมัติหรือหักยอดวันลาจนกว่า required fields จะครบ

## Approved Control Decisions

| Decision ID | Approved Outcome                                                                                                                                                                           |
| :-----------: | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| DEC-HR-001  | Payroll Run ต้องใช้ HR maker, Manager A1 และ Admin A2 สำหรับ privileged/org-wide run; Posted immutable; payslip เป็น owner-only; Leave Request แยก requester/approver และยอดต้อง reconcile |

- Payroll period เดียวกันสร้าง active run ได้เพียงหนึ่งรายการต่อ legal entity
  และ currency; correction ต้องอ้างอิง Posted run เดิม
- Calculation ต้อง snapshot earning/deduction, tax, social security, provident
  fund rule/version และ effective date ที่ใช้
- Admin A2 ต้องเป็นคนละบัญชีกับ HR maker และ Manager A1
- Leave policy ต้องมี version/effective date, accrual/carry/expiry rule,
  working-day calendar, overlap rule และ payroll effect

## Required Information

### Payroll Run

- Payroll Run ID, period, pay date และ status
- Legal entity, currency, calculation rule/version และ effective date
- Employee ID, base pay, OT hours/rate, เบี้ยขยัน และโบนัส
- ภาษีหัก ณ ที่จ่าย, เงินสมทบประกันสังคม และกองทุนสำรองเลี้ยงชีพ
- Gross pay, total deduction, net pay, control totals, payslip reference และ
  export batch ID

### Leave Request

- Leave Request ID, Employee ID, leave type, start/end date และ balance
  ก่อน/หลัง
- Leave policy/version, working-day calendar, requested units และ overlap result
- Requester, approver, เหตุผล และ attachment เมื่อจำเป็น (เช่น ใบรับรองแพทย์)
- Status และ approval timestamp

## Procedure

### 1. Prepare and Calculate Payroll Run

**Responsible:** HR

1. เปิดรอบเงินเดือนใหม่ตาม pay period และตรวจ unique active run ต่อ legal
   entity/currency
2. ดึงข้อมูล base pay, OT, เบี้ยขยัน และโบนัสของพนักงานที่ active
3. คำนวณภาษีหัก ณ ที่จ่าย ประกันสังคม และกองทุนตาม master data/version ที่มีผล
4. ตรวจ gross, deduction, net pay และ employee/control totals ให้ reconcile
5. บันทึกสถานะ Calculated พร้อม input/config/output snapshot และ checksum

**Expected result:** Payroll run มีตัวเลขครบและพร้อมส่งอนุมัติ

### 2. Approve and Post Payroll Run

**Responsible:** Manager (A1), Admin (A2 สำหรับ privileged/ทั้งองค์กร)

1. Manager ตรวจยอดรวมเทียบกับ budget/headcount
2. รอบเงินเดือนแบบ privileged หรือทั้งองค์กรต้องมี Admin A2 อนุมัติเพิ่ม
   โดยเป็นคนละบัญชีจาก HR ผู้จัดทำและ Manager A1
3. อนุมัติแล้วเปลี่ยนสถานะเป็น Approved และ lock ตัวเลข
4. ระบบ Post ด้วย idempotency key เข้าสถานะ Posted พร้อมสร้าง payslip และ
   accounting export batch
5. การแก้ไขหลัง Posted ต้องสร้าง correction run แยกต่างหาก ห้ามเขียนทับ

**Expected result:** Payroll run ผ่านการอนุมัติแยกจากผู้จัดทำ และมี audit trail
ครบ

### 3. Issue Payslip

**Responsible:** ระบบ (อัตโนมัติหลัง Posted), HR (แก้ไขกรณีข้อผิดพลาด)

1. สร้าง payslip ออนไลน์ต่อพนักงานหลัง payroll run เป็น Posted
2. พนักงานเข้าดู payslip ของตนเองเท่านั้นผ่านบัญชีของตน
3. ข้อผิดพลาดใน payslip ต้องแก้ผ่าน correction run ห้ามแก้ไฟล์โดยตรง

**Expected result:** พนักงานเข้าถึง payslip ของตนเองได้ และข้อมูลตรงกับ payroll
run ที่ Posted

### 4. Submit and Approve Leave Request

**Responsible:** ผู้ยื่นคำขอ (self-service) หรือ HR, Manager หรือ HR (Approver)

1. พนักงานยื่นคำขอลาระบุประเภท ช่วงเวลา หน่วยลา และเหตุผล
2. ระบบตรวจ active policy/version, working-day calendar, overlap และ balance
3. Manager อนุมัติคำขอของทีมตนเอง หรือ HR อนุมัติเมื่อไม่มี Manager
   โดยตรงหรือเป็นคำขอมาตรฐาน
4. อนุมัติแล้วหักยอดวันลาคงเหลือแบบ atomic และบันทึก before/after event
5. ยกเลิกคำขอที่อนุมัติแล้วต้องผ่าน authority ตาม policy คืนยอดแบบ atomic
   และสร้าง reversal event
6. เมื่อถึงวันลา ระบบเปลี่ยน Approved เป็น Taken และส่ง payroll effect ที่
   policy กำหนดด้วย idempotency key

**Expected result:** คำขอลาผ่านการอนุมัติที่แยกจากผู้ยื่น และยอดวันลาถูกต้อง

### 5. Export Payroll Data to Accounting

**Responsible:** HR

1. สร้าง export batch หลัง payroll run เป็น Posted โดยอ้างอิง
   run/version/checksum
2. Accounting export สำหรับ Finance ต้องเป็นยอดรวมและ account mapping ที่ไม่มี
   Employee ID, ชื่อ, payslip หรือค่าจ้างรายบุคคล และใช้ Manager A1 อนุมัติ
3. Individual/statutory export จัดเป็น Restricted ใช้ purpose, authorized
   recipient, Manager A1 + Admin A2, expiry และ secure delivery
4. บันทึก export log พร้อม classification, requester, approvers, recipient,
   checksum, เวลา และ delivery result

**Expected result:** ข้อมูลเงินเดือนถึงทีมบัญชีครบถ้วนและตรวจสอบย้อนหลังได้

## Control Points

| Control                | Requirement                                                                        | Evidence                                      |
| ---------------------- | ---------------------------------------------------------------------------------- | --------------------------------------------- |
| Payroll maker-checker  | HR maker, Manager A1, Admin A2 เมื่อ privileged/org-wide และ System post แยก actor | Prepare/A1/A2/post events และ SoD result      |
| Period/config snapshot | unique active run และเก็บ calculation/master version ที่ใช้                        | Period constraint, input/config/output hash   |
| Payslip access         | พนักงานเห็นเฉพาะ payslip ของตนเอง                                                  | Employee-bound authorization และ access log   |
| Leave approval         | ผู้ยื่นห้ามอนุมัติเอง; balance/overlap/reversal ต้อง atomic                        | Requester/approver และ balance events         |
| Finance isolation      | Finance เห็นเฉพาะ accounting aggregate ที่ไม่มีข้อมูลเงินเดือนรายบุคคล             | Export schema/classification และ denial event |
| Posted immutability    | แก้ไขหลัง Posted ต้องใช้ correction run ซึ่งอ้างอิง original                       | Original/correction event และ reconciliation  |

## Exceptions

- **Payroll ผิดพลาดหลัง Posted:** สร้าง correction run แยก ห้ามเขียนทับรอบเดิม
- **Post/retry ซ้ำ:** คืนผลเดิมจาก idempotency key ห้ามสร้าง payslip/export ซ้ำ
- **Leave balance ไม่พอ:** ปฏิเสธ หรือส่งขออนุมัติพิเศษพร้อมเหตุผล
- **คำขอลาซ้อน:** block จนยกเลิกรายการเดิมหรือผ่าน documented exception
- **พนักงานลาออก/สถานะเปลี่ยนกลางรอบ:** คำนวณตามสัดส่วนวันทำงานจริงและปิดยอด
  วันลาคงเหลือ
- **Manager ไม่ acknowledge คำขอลาในเวลา:** ยกระดับให้ HR อนุมัติแทนตาม policy
