## Purpose

กำหนดวิธีสร้าง ตรวจ ส่งออก และทบทวน Report/Audit Log ให้ข้อมูลถูกต้องตาม as-of
time, permission, Tenant และ source version พร้อมควบคุมไฟล์ส่งออก

## Scope

ครอบคลุม Matter/Client, Task/Calendar, Time/Productivity, Quotation, Billing,
Finance, HR, system/security report, dashboard, Excel/PDF/CSV export, scheduled
report, audit review และ anomaly follow-up

## Roles

| Role             | Responsibility                                                                                           |
| ---------------- | -------------------------------------------------------------------------------------------------------- |
| Report Requester | ระบุ purpose, scope, filters และ recipient                                                               |
| Data Owner       | ตรวจความหมาย field, completeness และ reconciliation                                                      |
| Manager          | อนุมัติ sensitive/management export และใช้ผลตัดสินใจ                                                     |
| Finance          | Validate financial report และเห็นเฉพาะ finance scope                                                     |
| HR               | Validate HR/payroll report และเห็นเฉพาะ HR scope                                                         |
| Admin            | ดู system/audit report, ดูแล report configuration, อนุมัติ Restricted/Privileged export และ review audit |

Finance ห้ามเห็น payroll รายบุคคล และ HR ห้ามเห็น Matter/billing data นอก
permission ที่อนุมัติ

## Required Report Metadata

- Report ID/name, definition/version และ data sources
- Tenant ID, requester, purpose และ permission scope
- Filters, period, timezone, as-of/generated at และ data freshness
- Currency/conversion rule เมื่อรวมหลายสกุลเงิน
- Row count, totals/control totals และ reconciliation result
- Classification, format, recipient, expiry และ approval เมื่อ export

## Procedure

### 1. Request and Generate a Report

**Responsible:** Report Requester

1. เลือก approved report definition และระบุ purpose/period/filters
2. ระบบบังคับ Tenant, role, Matter/team และ field-level permission
3. ระบุ timezone, as-of time และ currency rule
4. Generate จาก source records ที่มีผลและเก็บ definition/source versions
5. แสดง freshness และข้อจำกัดของข้อมูลบน report

**Expected result:** Report มีขอบเขตและเวลาอ้างอิงชัดเจน

### 2. Validate and Reconcile

**Responsible:** Data Owner

1. ตรวจ required source, duplicate/missing record และ status mapping
2. Reconcile row count, opening/closing balance และ control totals ตาม domain
3. Financial report ต้อง reconcile กับ Invoice/Payment/adjustment records
4. Workflow report ต้อง reconcile กับ immutable status events
5. ระบุ variance, materiality, owner และ resolution
6. ห้าม publish final report เมื่อ material variance ยังเปิด

**Expected result:** Report มี validation/reconciliation result
ที่ตรวจย้อนกลับได้

### 3. Export or Schedule Delivery

**Responsible:** Report Requester, Manager/Admin (Approver)

1. ระบุ format Excel/PDF/CSV, recipient, purpose และ expiry
2. ตรวจ classification, row/field scope และ need-to-know
3. Sensitive export ใช้ Manager approval; Restricted/Privileged ใช้ Manager A1
   และ Admin A2
4. สร้างไฟล์จาก validated report พร้อม checksum/watermark เมื่อเกี่ยวข้อง
5. ส่งผ่าน approved secure channel และ expiring access
6. Scheduled report ต้อง re-check permission ทุกครั้งก่อนส่ง
7. เก็บ download/delivery/failure event และ revoke เมื่อหมดอายุ

**Expected result:** Export ส่งถึงผู้รับที่อนุมัติและมีอายุจำกัด

### 4. Review Audit and Access Events

**Responsible:** Admin (Prepare), Admin คนละคน (Review)

1. ตรวจ login, permission, Emergency Access, sensitive view/export, approval,
   correction และ disposal events
2. ทบทวน privileged/security events อย่างน้อยรายเดือน
3. ตรวจ event sequence, missing event, integrity verification และ timestamp
4. เปรียบเทียบ access กับ active role/permission และ recertification result
5. ระบุ anomaly, owner, severity, due date และ incident/reference ID
6. Manager รับทราบ material finding และติดตาม remediation

**Expected result:** Audit review มี scope, reviewer, finding และ remediation

### 5. Correct a Report or Audit Finding

**Responsible:** Data Owner หรือ Admin

1. ห้ามแก้ source/audit event เพื่อทำให้ report ตรง
2. แก้ source ผ่าน approved correction/adjustment process
3. Generate report ใหม่ด้วย version/as-of ใหม่
4. Link superseded report/export และแจ้ง recipient เมื่อผลเปลี่ยน materially
5. Audit finding ปิดได้เมื่อมี evidence และ reviewer ยืนยัน

**Expected result:** Correction รักษา original report/event และแสดง lineage
ชัดเจน

## Control Points

| Control             | Requirement                                               | Evidence                                      |
| ------------------- | --------------------------------------------------------- | --------------------------------------------- |
| Tenant/row security | Report/query/export ใช้ permission เท่ากับ source         | Authorization decision และ filtered row count |
| Definition version  | Metric/filter/mapping ต้องมี version/effective date       | Report definition/version                     |
| Reconciliation      | Material variance ต้องปิดก่อน final publish               | Control totals และ reconciliation result      |
| Sensitive export    | Manager approval; Restricted/Privileged ใช้ A1/A2         | Export request, approvals และ SoD             |
| File protection     | Checksum, encryption, watermark/expiry ตาม classification | Export metadata และ delivery event            |
| Scheduled delivery  | Re-check permission/recipient ทุกครั้ง                    | Schedule run authorization                    |
| Audit integrity     | Audit event ห้ามแก้/ลบและต้องตรวจ sequence/integrity      | Verification result และ correction linkage    |
| Retention           | Report lineage ตาม source; export log เก็บตาม SOP-AUD-001 | Policy version และ disposal eligibility       |

## Exceptions

- **ข้อมูลไม่สด:** แสดง freshness และห้ามใช้เป็น final เมื่อเกิน policy
- **Reconciliation ไม่ผ่าน:** ระงับ publish/export และ assign owner
- **Recipient ไม่มี permission:** ห้ามส่ง แม้ requester มีสิทธิ์
- **Scheduled recipient เปลี่ยน role:** หยุด delivery และแจ้ง owner
- **Export ผิดผู้รับ:** revoke link, เปิด incident และแจ้ง Admin
- **Audit integrity fail:** ระงับ disposal และเปิด security incident
- **Source correction:** generate report ใหม่ ห้ามแทนที่ไฟล์/ผลเดิม
