## Purpose

กำหนดวิธีตั้งค่าและส่ง communication/notification อย่างควบคุม เชื่อมประวัติกับ
Client/Matter โดยไม่รั่วข้อมูล และรับ ติดตาม แก้ไข/escalate support ticket พร้อม
ดูแล Knowledge Base/Chatbot response ให้มี owner, version และหลักฐาน

## Scope

ครอบคลุม email connection handoff, notification template/delivery/preferences,
case alert, Admin broadcast, inbound/outbound communication log, Client/Matter
link/search, helpdesk ticket, classification/priority/SLA, assignment,
resolution/closure, knowledge article, chatbot answer/source/escalation,
support report และ workflow synchronization

ไม่ครอบคลุม business trigger/decision ใน SOP domain, provider/API credential ใน
SOP-INT-001, P1-P4 exception/incident authority ใน CTRL-EXC-001 หรือ Client Portal
publication ใน SOP-CLI-002 Support ticket ไม่อนุญาตให้แก้ business/financial/
security record โดยข้าม SOP เจ้าของข้อมูล

## Roles

| Role                  | Responsibility                                                                            |
| --------------------- | ----------------------------------------------------------------------------------------- |
| Service Support Owner | เป็น process owner ดูแล channel/template/ticket/knowledge/SLA และ support reporting        |
| Support Agent         | รับ/จัดประเภท/ตอบ/ติดตาม ticket และ communication ตาม scope                                |
| Communication Owner   | ยืนยัน purpose/audience/content/template และ business trigger                               |
| Domain Owner          | แก้ business issue/อนุมัติข้อความของ domain และยืนยัน resolution                            |
| Knowledge Owner       | จัดทำ/ทบทวน article และ approved chatbot source                                             |
| Manager A1            | อนุมัติ high-impact broadcast, sensitive template และ exception/resolution                  |
| Admin A2              | อนุมัติ platform-wide/privileged channel/config และแยกจาก executor                          |
| Admin/System          | configure/send/route/retry/suppress หลัง approval และเก็บ immutable events                  |
| User/Client Contact   | รับ communication, ตั้ง preference ตาม policy, เปิด ticket และยืนยันผล                       |

## Approved Status Baseline

### Communication Delivery

Queued, Sent, Delivered, Failed, Suppressed และ Cancelled

### Support Ticket

New, Acknowledged, In Progress, Waiting User, Escalated, Resolved, Closed และ
Cancelled

### Knowledge Article

Draft, In Review, Approved, Published, Superseded และ Archived

- Sent ไม่เท่ากับ Delivered
- Resolved ยังไม่เท่ากับ Closed จน validation/user confirmation/closure rule ครบ
- Escalated ticket ที่เป็น P1-P4 ใช้ CTRL-EXC-001 โดยไม่ reset SLA clock
- Chatbot ใช้เฉพาะ Published source และต้องเปิด human handoff

## Required Information

- Channel/Connection reference, owner, Tenant/environment, provider connection ID
  จาก SOP-INT-001 และ health status
- Template/Message ID/version, trigger/source record, purpose, classification,
  audience/recipient, language, variables, preference/consent, schedule/expiry
- delivery/correlation/idempotency/provider IDs, queued/sent/delivered/failed time,
  retry/suppression/cancellation reason
- Communication Log ID, direction/channel/participants, Client/Matter linkage,
  subject/classification, content reference/checksum และ access scope
- Ticket ID/source/requester, Tenant, affected record/service, category/priority,
  impact, status, owner, SLA clocks, communication และ resolution
- Article ID/version/owner/source/review/expiry; chatbot query/source/answer/
  confidence-or-limitation/handoff/feedback
- A1/A2/executor/domain/knowledge actors และ exception/change references

## Procedure

### 1. Configure Channel and Notification Template

**Responsible:** Communication Owner, Service Support Owner, Admin/System

1. อ้าง Active connection จาก SOP-INT-001 ห้ามเก็บ raw credential ใน template
2. สร้าง template ระบุ purpose/trigger/audience/language/variables/classification
3. ตรวจ content minimization, safe subject/preview, preference/consent และ expiry
4. sensitive/high-impact ใช้ Manager A1; platform-wide/config ใช้ Admin A2
5. test synthetic recipients/invalid variable/suppression/failure ก่อน Published

**Expected result:** approved template/version ส่งข้อมูลเท่าที่จำเป็น

### 2. Queue, Send and Track Notification

**Responsible:** Domain System, Support/System

1. รับ approved business trigger/source ID จาก SOP domain
2. ตรวจ current template/version, recipient scope, preference/consent และ expiry
3. render โดยไม่ใส่ restricted content ใน email/SMS/push preview; ใช้ secure link
4. queue/send ด้วย correlation/idempotency key และเก็บ provider acknowledgment
5. status เป็น Delivered เฉพาะ evidence รองรับ; bounce/fail/suppress เก็บเหตุผล

**Expected result:** delivery state ตรงจริงและ retry ไม่ส่งซ้ำโดยไม่ควบคุม

### 3. Send Case Alert or Admin Broadcast

**Responsible:** Communication Owner, Manager A1/Admin A2, System

1. ระบุ urgency, exact audience/filter, purpose, content, schedule และ expiry
2. case alert ใช้ Matter permission; Admin broadcast ห้ามแนบ business content
3. high-impact/external/sensitive ใช้ A1; platform-wide/emergency ใช้ A1+A2
4. preview audience count/sample แล้ว approve exact version ก่อน queue
5. reconcile intended/queued/sent/delivered/failed/suppressed/cancelled totals

**Expected result:** broadcast ถึง audience ที่อนุมัติและตรวจยอดได้

### 4. Record and Link Communication

**Responsible:** User/Support Agent, System

1. capture inbound/outbound metadata, participants, time/channel/direction และ
   content reference/checksum
2. link Client/Matter เมื่อ authorization และ identity match
3. classify content/attachment และจำกัด search/list/preview/export
4. correction สร้าง linked revision ห้ามแก้ finalized communication log
5. retention/Legal Hold ใช้ source/domain policy และ SOP-AUD-001

**Expected result:** communication ค้นย้อนกลับได้โดยไม่รั่ว content/metadata

### 5. Receive, Classify and Acknowledge Ticket

**Responsible:** User/Client Contact, Support Agent

1. รับ ticket จาก portal/email/form/chatbot/agent พร้อม Tenant/requester/channel
2. ตรวจ duplicate/related incident และ redacted description/attachments
3. classify service/domain/category/impact/priority และเริ่ม SLA clock
4. acknowledge พร้อม Ticket ID, status, next update โดยไม่รับรองผลก่อนตรวจ
5. P1-P4/security/privacy/deadline/financial issue เปิด CTRL-EXC-001/domain handoff

**Expected result:** ticket มี owner/SLA/scope และ escalation ที่ถูกต้อง

### 6. Investigate, Resolve and Close Ticket

**Responsible:** Support Agent, Domain Owner, User

1. Support Agent ตรวจ knowledge/log/known issue ด้วย least-privilege support access
2. business/data/config correction ส่ง SOP/domain owner ห้ามแก้ production ตรง
3. บันทึก diagnosis, action/change/incident, workaround/expiry และ communication
4. Domain Owner validate result; user confirm หรือ closure rule พร้อม evidence
5. Resolved → Closed เมื่อ no orphan/variance, notice และ follow-up ครบ

**Expected result:** resolution ตรวจย้อนกลับได้และไม่ใช้ ticket แทน approval

### 7. Create and Publish Knowledge Article

**Responsible:** Knowledge Owner, Domain Owner, Manager

1. Draft article ระบุ audience, problem, prerequisites, steps, limitations,
   owner/source/version/review/expiry
2. ตัด secret/personal/client-specific/internal exploit detail ตาม classification
3. Domain Owner ตรวจความถูกต้อง; sensitive/policy article ใช้ Manager approval
4. publish exact version; update ใช้ new version และ Superseded เดิม
5. stale/invalid article Archive/withdraw และ link replacement

**Expected result:** user/agent/chatbot ใช้ approved current knowledge

### 8. Operate Chatbot and Human Handoff

**Responsible:** Service Support/Knowledge Owner, System

1. chatbot ค้นเฉพาะ Published article/source ที่ audience มีสิทธิ์
2. answer แสดง source/version/limitation และไม่สมมติ business/legal decision
3. low confidence, no source, personal/sensitive, complaint หรือ action request
   ต้องส่ง human/ticket พร้อม context ที่จำเป็น
4. ห้าม chatbot post/change/approve/reveal record โดยไม่มี authorized workflow
5. เก็บ query classification/source/result/handoff/feedback โดยลด personal content

**Expected result:** chatbot เป็น support interface ไม่ใช่ decision authority

### 9. Review Service Performance and Improve

**Responsible:** Service Support Owner, Domain/Knowledge Owners

1. reconcile delivery/ticket/SLA/reopen/escalation/article/chatbot handoff metrics
2. sample failed/suppressed/duplicate/overdue/closed evidence และ access scope
3. recurring issue เปิด problem/corrective action หรือปรับ knowledge/template
4. report ใช้ SOP-REP-001 พร้อม as-of/filter/source version
5. review permission/templates/articles/connections/retention ตามรอบ

**Expected result:** service improvement อ้าง evidence และไม่มี silent backlog

## Control Points

| Control                       | Requirement                                                                  | Evidence                                               |
| ----------------------------- | ---------------------------------------------------------------------------- | ------------------------------------------------------ |
| Approved channel/template     | Active connection, owner/purpose/audience/version/classification/variables     | template/config approval and test result               |
| Metadata-safe delivery        | subject/preview/link/variables ไม่เผยข้อมูลเกิน recipient scope                | rendered sample and authorization result               |
| Delivery truth/idempotency    | Sent/Delivered/Failed/Suppressed ตาม evidence; retry ไม่ส่งซ้ำ                 | correlation/provider IDs and delivery events           |
| Audience reconciliation       | intended/queued/sent/delivered/failed/suppressed/cancelled totals ตรง           | broadcast manifest and control totals                  |
| Communication access/history | Client/Matter link, classification, search/export permission และ immutable revision | log linkage/access/revision events                  |
| Ticket SLA/domain handoff     | category/priority/owner/clocks/escalation ไม่ reset และแก้ผ่าน SOP owner       | ticket/exception/change/domain references              |
| Resolution/closure evidence   | validation/user confirmation/no orphan/variance และ communication              | resolution test, acceptance and closure event          |
| Knowledge/chatbot governance  | approved source/version/audience/limitation/expiry และ human handoff            | article approval and chatbot source/handoff events     |

## Exceptions

- **Recipient/scope ไม่ถูกต้อง:** suppress/cancel ก่อนส่ง; หากส่งแล้ว revoke link,
  preserve evidence และประเมิน privacy incident
- **Provider/connection fail:** status Failed, retry/backoff ผ่าน SOP-INT-001;
  ห้าม mark Delivered
- **Duplicate notification:** idempotency/correlation คืนผลเดิม; reconcile totals
- **Preference/consent conflict:** suppress และส่ง Data/Privacy Owner review
- **Ticket เป็น P1-P4:** link CTRL-EXC-001 โดยคง original ticket/SLA clocks
- **Support ต้องแก้ production:** เปิด approved domain/change workflow ห้ามใช้
  support identity แก้ตรง
- **Knowledge stale/wrong:** withdraw/supersede แจ้ง affected tickets/users
- **Chatbot ไม่มี approved source/พบ sensitive request:** ไม่ตอบเนื้อหา ส่ง human
