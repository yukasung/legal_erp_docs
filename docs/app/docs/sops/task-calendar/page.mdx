## Purpose

กำหนดวิธีสร้าง มอบหมาย ติดตาม และปิด Task รวมถึงสร้างนัดหมาย แจ้งเตือน
จัดการตารางทีม และซิงก์ปฏิทินโดยไม่เปิดเผยข้อมูลเกิน permission

## Scope

ครอบคลุม Task List/Board, assignment, priority, due date, status, notes,
attachment, notification, overdue/workload และ Appointment List, recurring,
attendance, reminder, availability, conflict override และ calendar integration

## Roles

| Role      | Responsibility                                                    |
| --------- | ----------------------------------------------------------------- |
| Lawyer    | สร้าง/ทำ Task และจัดนัดหมายของ Matter ที่รับผิดชอบ                |
| Assistant | เตรียม มอบหมาย ติดตาม reminder และอัปเดตตาม permission            |
| Manager   | จัด workload, approve conflict override และ completed edit        |
| Finance   | ทำ Task/Calendar ทางการเงินที่ได้รับมอบหมาย                       |
| Admin     | ตั้ง category, status, notification และ integration configuration |

## Approved Status Baseline

### Task

Draft, Open, In Progress, Waiting, Blocked, Completed และ Cancelled

### Appointment

Draft, Scheduled, Completed, Cancelled, No-show และ Rescheduled

Draft ห้ามส่ง assignment/reminder จนกว่า required fields และ validation ผ่าน

## Required Information

### Task

- Task ID, title, description, category, priority และ status
- Matter ID/Client ID เมื่อเกี่ยวข้อง
- Assignee, assigner, start/due date และ timezone
- Attachment Document IDs, dependency, reminder และ confidentiality

### Appointment

- Appointment ID, title, start/end, timezone และ status
- Matter ID/Client ID, organizer และ participants เมื่อเกี่ยวข้อง
- location/meeting link, recurrence, reminder, attachments และ notes
- calendar mapping และ external event ID เมื่อ sync

## Procedure

### 1. Create and Assign a Task

**Responsible:** Lawyer, Assistant หรือ Manager

1. ค้นหา Task เดิมและตรวจ Matter/Client permission
2. ระบุ title, category, priority, start/due date และ assignee
3. ตรวจว่า assignee เป็น active มี permission และ capacity ตาม policy
4. เชื่อม attachment ด้วย Document ID และรับ confidentiality ที่เข้มงวดที่สุด
5. บันทึก Draft เมื่อข้อมูลไม่ครบ หรือ Open เมื่อ validation ผ่าน
6. ส่ง assignment notification หลัง Task เป็น Open เท่านั้น

**Expected result:** Task มี owner, deadline, linkage และ notification
ที่ถูกต้อง

### 2. Update and Complete a Task

**Responsible:** Assignee หรือผู้มี assign permission

1. อัปเดต Open เป็น In Progress, Waiting หรือ Blocked ตามสถานการณ์
2. Waiting/Blocked ต้องมี reason, dependency owner และ review/due date
3. บันทึก progress note พร้อม actor/timestamp
4. Reassign ต้องระบุ reason และตรวจ assignee ใหม่
5. Completed ต้องมี completion note และ completed at
6. การแก้ Completed Task ต้องสร้าง revision และ Manager approval
7. Bulk update ทำได้เมื่อผู้ใช้มี permission กับทุกรายการและต้องรายงานผลราย Task

**Expected result:** Task history สะท้อน owner, status และความคืบหน้าจริง

### 3. Monitor Deadlines and Workload

**Responsible:** Assignee, Manager

1. ระบบส่ง due reminder ตาม Tenant Policy
2. เมื่อพ้น due date ให้ระบุ Overdue โดยไม่เปลี่ยน status เป็น Completed
3. แจ้ง assignee/assigner และ escalate Manager ตาม priority policy
4. Manager ตรวจ workload, overdue และ overload flag ก่อน reassign
5. คำแนะนำ assignee ของระบบเป็น recommendation ห้าม reassign อัตโนมัติ

**Expected result:** งานใกล้ครบกำหนด/เกินกำหนดมี owner และ escalation ชัดเจน

### 4. Create or Update an Appointment

**Responsible:** Lawyer หรือ Assistant

1. ระบุ title, start/end, timezone, organizer, participants และ linkage
2. ตรวจ start ก่อน end, participant availability และ scheduling conflict
3. Recurring appointment ต้องมี pattern, start และ end/count
4. Draft ห้าม trigger reminder
5. Scheduled ต้องมี reminder ก่อนเวลา appointment และช่องทางที่อนุมัติ
6. การแก้ recurring series ต้องระบุ single occurrence หรือ entire series
7. Cancel ต้องมี reason และหยุด future reminder

**Expected result:** Appointment ถูกต้องตามเวลาและแจ้งผู้เกี่ยวข้องตาม
permission

### 5. Resolve Scheduling Conflicts

**Responsible:** Organizer (Requester), Manager (Approver)

1. ระบบตรวจ appointment และ blocked slot ที่ซ้อนกัน
2. เปลี่ยนเวลา/assignee เมื่อแก้ conflict ได้
3. หากต้อง override ให้ระบุ reason, impact และผู้เข้าร่วมที่รับทราบ
4. Manager อนุมัติก่อนบันทึก overlap
5. สร้าง conflict/override event และแจ้ง participant

**Expected result:** ตารางซ้อนมีเหตุผล ผู้อนุมัติ และ audit trail

### 6. Sync an External Calendar

**Responsible:** User เจ้าของ Calendar, Admin (Configuration)

1. เชื่อม Google/Outlook ผ่าน OAuth และขอ scope เท่าที่จำเป็น
2. เก็บ token แบบเข้ารหัสและไม่แสดงใน UI/log
3. กำหนด internal/external calendar mapping และ source of truth
4. Sync เฉพาะ field ที่ permission อนุญาตและไม่ส่ง privileged note โดย default
5. Conflict/error ต้องสร้าง sync event และรอ reconciliation
6. Disconnect ต้อง revoke token และคง history

**Expected result:** Calendar sync มี mapping, permission และ reconciliation

## Control Points

| Control              | Requirement                                         | Evidence                               |
| -------------------- | --------------------------------------------------- | -------------------------------------- |
| Valid assignment     | Assignee active, permitted และผ่าน capacity check   | User/status/permission validation      |
| Date/time validation | Start ก่อน due/end และใช้ Tenant timezone           | Validation result                      |
| Matter access        | List/detail/attachment กรองตาม Matter permission    | RBAC decision และ access event         |
| Completed edit       | ใช้ revision และ Manager approval                   | Previous/new values และ approval       |
| Reminder delivery    | เก็บ scheduled/sent/failed timestamp และ channel    | Notification event                     |
| Conflict override    | Manager อนุมัติพร้อม reason                         | Conflict result และ override approval  |
| External sync        | OAuth least scope, encryption และ reconciliation    | Connection/mapping/sync events         |
| Immutable history    | Assignment/status/note/attendance history read-only | Event chain และ integrity verification |

## Exceptions

- **Assignee inactive/ไม่มี permission:** ห้าม assign และเลือกผู้รับผิดชอบใหม่
- **Task Blocked:** ระบุ dependency owner และ review date
- **Task Overdue:** แจ้งและ escalate ห้าม mark Completed อัตโนมัติ
- **Appointment conflict:** เปลี่ยนเวลา หรือขอ Manager override
- **Reminder failed:** retry ตาม policy และแจ้ง organizer เมื่อยังล้มเหลว
- **External sync conflict:** คง internal record และ reconcile ก่อน overwrite
- **Matter On Hold/Closed:** ตรวจว่ารายการใหม่ได้รับอนุญาตตาม SOP-MAT-002
