## 1. สมัครและเปิดใช้งาน

สร้าง Workspace ที่พร้อมให้ทีมเริ่มทำงาน

```mermaid
flowchart TD
  A["1.1 สร้างบัญชี"] --> B["1.2 เลือกแผนใช้งาน"]
  B --> C{"แผนที่เลือกต้องชำระเงินหรือไม่"}
  C -->|ไม่ต้องชำระ| D["เปิด Workspace"]
  C -->|ต้องชำระ| E["ชำระเงินตามแผน"]
  E --> F{"ชำระเงินสำเร็จหรือไม่"}
  F -->|ไม่สำเร็จ| G["แก้ไขข้อมูลและชำระใหม่"]
  G --> E
  F -->|สำเร็จ| D
  D --> H["เชิญทีมและกำหนดสิทธิ์"]
  H --> R["ผลลัพธ์: Workspace พร้อมใช้งาน"]
```

### คำอธิบายขั้นตอน

1. **สร้างบัญชี:** ผู้สมัครกรอกข้อมูลสำหรับเข้าสู่ระบบและยืนยันอีเมล
   ผู้สมัครจะเป็นเจ้าของ Workspace และเป็นสมาชิกคนแรก
2. **เลือกแผนใช้งาน:** เลือกแผนฟรีหรือแผนเสียเงินตาม Module
   ที่ต้องการใช้
3. **ตรวจว่าต้องชำระเงินหรือไม่:** แผนฟรีเปิดใช้งานได้โดยไม่ต้องกรอกข้อมูลบัตร
   ส่วนแผนเสียเงินต้องชำระเงินก่อน
4. **ชำระเงินตามแผน:** ระบบส่งผู้สมัครไปยังหน้าชำระเงินที่ปลอดภัย
   และไม่เก็บหมายเลขบัตรหรือรหัสความปลอดภัยของบัตร
5. **กรณีชำระเงินไม่สำเร็จ:** ระบบยังไม่เปิดแผนเสียเงิน ผู้สมัครต้องแก้ไข
   ข้อมูลการชำระเงินแล้วลองใหม่
6. **เปิดใช้งาน Workspace:** ผู้สมัครทุกคนจะได้รับ Workspace เหมือนกัน
   ผู้สมัครที่เลือกใช้งานฟรีเปิดใช้งานได้ทันที ส่วนผู้สมัครแบบชำระเงิน
   เปิดใช้งานหลังระบบยืนยันการชำระเงินสำเร็จ
7. **เชิญทีมและกำหนดสิทธิ์:** เจ้าของ Workspace เชิญพนักงานเข้าระบบ
   และกำหนดว่าแต่ละคนเข้า Module หรือทำรายการใดได้
8. **ผลลัพธ์:** Workspace พร้อมใช้งาน ผู้สมัครสามารถใช้งานคนเดียว
   หรือเพิ่มสมาชิกภายหลังได้

## 3. ประเมินการรับงาน

ตรวจความซ้ำซ้อน ผลประโยชน์ขัดกัน และความเสี่ยง

```mermaid
flowchart TD
  A["3.1 สร้าง Matter สำหรับตรวจสอบก่อนรับงาน"] --> B["3.2 ตรวจว่ามี Matter นี้อยู่แล้วหรือไม่"]
  B --> C{"3.3 ตรวจ Conflict และประเมิน Risk"}
  C -->|ผ่าน| E["อนุมัติรับงาน"]
  C -->|ต้องทบทวน| F["Manager ตรวจเหตุผลและมาตรการ"]
  F --> G{"อนุมัติหรือไม่"}
  G -->|อนุมัติ| E
  G -->|ไม่อนุมัติ| H["ยุติการรับงานและเก็บเหตุผล"]
  C -->|ไม่ผ่าน| H
  E --> R["ผลลัพธ์: พร้อมจัดทำข้อเสนอ"]
```

## 4. ตกลงราคาและขอบเขต

## 5. เปิดและดำเนินงาน

ใช้ Matter เป็นศูนย์กลางของทีม งาน นัดหมาย เอกสาร และเวลา

```mermaid
flowchart TD
  A["5.1 เปิด Matter เป็น Active"] --> B["5.2 มอบหมาย Lawyer และทีม"]
  B --> C["5.3 ดำเนินงาน เอกสาร นัดหมาย และเวลา"]
  C --> D{"งานตามขอบเขตเสร็จหรือไม่"}
  D -->|เสร็จแล้ว| R["Manager ตรวจและอนุมัติงาน"]
  R --> Z(["สิ้นสุด: Matter เป็น Completed และพร้อมวางบิล"])
  D -->|ยังไม่เสร็จ| E{"ต้องทบทวนหรือพักงานหรือไม่"}
  E -->|ไม่ต้อง| C
  E -->|ต้องทบทวนหรือพักงาน| F["ตรวจ Conflict, Risk หรืออนุมัติ On Hold"]
  F --> G{"อนุมัติให้ดำเนินงานต่อหรือไม่"}
  G -->|อนุมัติ| C
  G -->|ไม่อนุมัติ| H(["สิ้นสุด: หยุดงานและส่งให้ Manager ตัดสินใจ"])
```

</section>

## 6. วางบิลและรับชำระ

ออกใบแจ้งหนี้ รับเงิน และยืนยันยอดก่อนส่งปิด Matter

```mermaid
flowchart TD
  A["6.1 ออกใบแจ้งหนี้"] --> B["6.2 หักเงินที่ Client ชำระล่วงหน้า (ถ้ามี)"]
  B --> C["6.3 บันทึกการรับชำระและตัดยอดใบแจ้งหนี้"]
  C --> D{"ยอดชำระและหลักฐานครบถ้วนถูกต้องหรือไม่"}
  D -->|ครบและถูกต้อง| E["ฝ่ายการเงินตรวจสอบและยืนยันว่าชำระครบแล้ว"]
  D -->|ยังชำระไม่ครบ| F["ติดตามยอดค้าง"]
  F --> C
  D -->|ยอดหรือหลักฐานไม่ตรง| G["ตรวจสาเหตุและปรับปรุงตามสิทธิ์"]
  G --> C
  E --> R["ผลลัพธ์: ส่งข้อมูลที่ยืนยันแล้วไป Closure Checklist"]
```

## 7. ปิด Matter และเก็บรักษาหลักฐาน

ตรวจความครบถ้วนก่อนจำกัดการทำรายการและเริ่มระยะเวลาจัดเก็บข้อมูล

```mermaid
flowchart TD
  A["7.1 ตรวจงาน เอกสาร เวลา และยอดเงิน"] --> B{"ข้อมูลสำหรับปิดงานครบและได้รับอนุมัติหรือไม่"}
  B -->|ยังไม่ครบ| C["แก้ไขรายการที่ยังไม่ครบ"]
  C --> A
  B -->|ครบและอนุมัติ| D["7.2 ปิด Matter และจัดเก็บข้อมูลตามระยะเวลาที่กำหนด"]
  D --> Z(["สิ้นสุดขั้นตอนปิด Matter"])

  D -. เมื่อครบระยะเวลาจัดเก็บ .-> E{"มีเหตุที่ต้องเก็บข้อมูลต่อหรือไม่"}
  E -->|มี| F["ระงับการทำลายและกำหนดวันตรวจใหม่"]
  F -->|เมื่อเหตุสิ้นสุด| E
  E -->|ไม่มี| G{"Manager และ Admin อนุมัติให้ทำลายครบหรือไม่"}
  G -->|ยังไม่ครบ| H["เก็บข้อมูลต่อและส่งอนุมัติใหม่"]
  H --> G
  G -->|อนุมัติครบ| I["ทำลายข้อมูลตามขอบเขตที่อนุมัติ"]
  I --> J["บันทึกหลักฐานการทำลาย"]
  J --> Y(["สิ้นสุดกระบวนการทำลายข้อมูล"])
```

> **หมายเหตุ:** การปิด Matter ไม่ทำลายข้อมูลทันที การทำลายจะเริ่มได้เมื่อครบ
> ระยะเวลาจัดเก็บ ไม่มีเหตุที่ต้องเก็บต่อ และได้รับอนุมัติครบตามที่กำหนด
> ผู้อนุมัติและผู้ดำเนินการทำลายต้องเป็นคนละบัญชี
